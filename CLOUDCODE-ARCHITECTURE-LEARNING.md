## 📚 CloudCode 核心架构

### 1. 入口层设计（启动与分流）

**核心逻辑**: 分级处理策略，简单命令与完整工作流分离

**短路径处理**:
- 针对简单命令（version、启动后台服务、远程控制）
- 直接在入口层完成并退出
- 实测数据：version 命令耗时 0.07 秒，内存占用 8.2 兆

**主程序加载**:
- 仅当需要进入持续交互时才加载完整系统
- 实测数据：help 命令耗时 0.44 秒，内存峰值 184 兆

**主程序模式分类**:
- print 模式：一次性输出结果后退出
- 交互模式：加载完整工作台（REPL 界面）
- 特殊入口：ssh 等专用场景处理

---

### 2. 交互工作台（REPL）

**定义**: REPL（Read-Eval-Print Loop）即输入-处理-输出循环界面

**触发条件**: 仅在交互模式下启动

**职责**:
- 接收用户输入
- 展示结果
- 权限确认

**定位**: 前台界面，不直接处理业务逻辑，仅负责任务分发

---

### 3. 输入处理与消息标准化

**本地命令过滤**:
- 优先判断是否为斜杠命令（如 /revert、/mcp）
- 非本地命令则进入正常处理流程

**消息对象包装**:
- 将用户输入转换为系统统一消息对象
- 确保后续模块采用一致格式处理

---

### 4. 上下文构建（核心数据准备）

**上下文由三部分组成**:

| 组成部分 | 具体内容 |
|---------|----------|
| **项目背景** | cloud.md 项目约定、记忆文件、当前日期 → 包装为 meta 插入消息最前端 |
| **系统提示词** | 默认规则（prompt.ts 文件）+ 用户附加规则 |
| **任务信息** | 当前消息、工具列表、MCP 连接、模型配置、状态中断处理逻辑 |
| **附加参数** | 历史消息、工具列表、模型设置、思考模式等 |

---

### 5. 核心处理模块（Corey）

**职责**:
- 任务接管：从 REPL 接收上下文
- 负责主循环执行
- 上下文压缩
- 模型调用
- 工具执行

**上下文压力处理**:
- 在调用模型前检查长度
- 触发 contacts collapse 或 auto compact 机制避免超限

**模型流式交互**:
- 实时判断输出类型
- 普通文本：直接展示给用户
- 工具调用信号：暂停文本输出，进入工具执行流程

---

### 6. 工具调用与权限控制

**工具执行流程**:

1. **权限判断**:
   - 模型仅能申请调用工具
   - 需通过权限系统验证（永久允许/拒绝/每次询问）

2. **并发安全控制**:
   - 可并发工具：文件读取等只读操作
   - 串行工具：bench/power shell 等需判断安全性

3. **结果循环处理**:
   - 工具执行结果返回至消息历史
   - 重新进入模型处理流程
   - 直至模型不再请求新工具

---

### 7. 复杂权限系统

**多层叠加验证机制**:

| 验证层级 | 说明 |
|---------|------|
| **全局权限表** | 预设工具永久允许/拒绝规则 |
| **工具自检** | 调用工具内置权限检查函数 |
| **模式改写** | 根据当前权限模式调整验证结果 |
| **分类器判断** | 尝试从其他渠道获取权限决策，减少用户交互 |
| **用户确认** | 最终需用户弹窗批准（如高风险操作） |
| **沙箱隔离** | 对 bench/power shell 等高危命令，在受限环境中执行 |

---

### 8. 专题一：上下文压缩策略

**核心目标**: 避免上下文超限，保障模型正常响应

**压缩触发机制**:
- 自动压缩：仅当上下文接近极限时触发，预留安全空间
- 分层压缩策略：
  a. 优先整合旧对话为记忆板历史（短文本摘要）
  b. 若仍超限，执行常规压缩（内容精简）

**错误恢复机制**:
- 模型返回超长错误时，先尝试自动恢复
- 若失败则执行应急压缩（reactive comp）

---

### 9. 专题二：权限系统设计亮点

**核心原则**: 模型无直接执行权，所有工具调用需通过多层安全验证

**高危命令特殊处理**:
- 对 bench/power shell 解析命令结构
- 检查操作路径
- 判断读写性
- 可能启用沙箱隔离

**安全设计哲学**: 通过规则校验、工具自检、模式控制、用户确认、沙箱隔离的多层叠加，确保执行权牢牢掌握在系统手中

```python
    return full_system_execute(cmd)
```

**优先级**: 🔴 高

### 2. 分层压缩策略 ⭐⭐⭐⭐

**价值**: 更精细的上下文管理

**实施建议**:
```python
# 分层压缩
if context_too_long():
    # 第一层：旧对话压缩
    compress_old_conversations()
    
    # 第二层：常规压缩
    if still_too_long():
        regular_compress()
    
    # 第三层：应急压缩
    if emergency():
        emergency_compress()
```

**优先级**: 🟡 中

### 3. 多层权限验证 ⭐⭐⭐⭐⭐

**价值**: 更安全的执行控制

**实施建议**:
```python
# 多层验证
def check_permission(tool, action):
    # 第 1 层：全局权限表
    if not global_permission_check(tool, action):
        return False
    
    # 第 2 层：工具自检
    if not tool_self_check(tool, action):
        return False
    
    # 第 3 层：模式改写
    if not mode_check(tool, action):
        return False
    
    # 第 4 层：分类器判断
    if not classifier_check(tool, action):
        return False
    
    # 第 5 层：用户确认
    if not user_confirm(tool, action):
        return False
    
    # 第 6 层：沙箱隔离
    return sandbox_execute(tool, action)
```

**优先级**: 🔴 高
