## 🎯 Claude Code 的 Multi-Agent 协作

### 核心定义
**一个 Agent 搞不定，拆成多个 Agent 并行干**

### 协作机制

#### 主 Agent 的限制
```
主 Agent 被剥夺所有动手能力
只剩 3 个工具：
1. 派出子 Agent
2. 给子 Agent 发消息
3. 叫停子 Agent
```

#### 子 Agent 的特点
- ✅ **并行工作** - 多个子 Agent 同时执行
- ✅ **互不知道** - 子 Agent 之间不知道对方存在
- ✅ **带工具** - 每个子 Agent 有完整的工具集

### 残酷规则

> **"子 Agent 用完即毁，只留结论。"**

**执行流程**:
```
干完活 → 子 Agent 消失
         ↓
    思考过程、中间步骤、纠结——全部丢弃
         ↓
         只保留最终结果
```

### 第三原理
**多 Agent 并行协作，用完即焚**

## 💡 关键差异

### Claude Code 的优势

1. **完全的"用完即毁"**
   - 子 Agent 完全消失
   - 思考过程全部丢弃
   - 节省记忆空间

2. **真正的并行**
   - 多个子 Agent 同时执行
   - 互不知道对方存在

### OpenClaw 的优势

1. **专业分工明确**
   - 小新（技术）
   - 小蓝（日志）
   - 设计专家（设计）
   - 每个专家有明确的职责

2. **沟通隔离严格**
   - 专业 Agent 不和用户直接沟通
   - 所有沟通通过大领导
   - 避免混乱

3. **编排系统完善**
   - 任务分析
   - 状态跟踪
   - 质量检查
   - 结果交付

4. **可追溯性**
   - 保留了子 Agent 的执行历史
   - 可以通过 sessions_history 查看
   - 便于调试和优化
