## 🎯 Claude Code 的记忆管理

### 核心定义
**Agent 的脑容量有限，记忆管理是第一约束**

### 残酷现实

```
Agent 的"工作记忆" ≈ 一本书
每说一句话就要从头翻一遍
随便一次对话 = 几万到十几万 Token
```

### 三套组合拳

#### 1️⃣ 缓存（Caching）
**不变的东西只记一次**

**优化策略**:
- 工具按字母表锁死排序
- 配置文件用内容哈希值命名
- 精准到改一个字就自动刷新缓存

#### 2️⃣ 压缩（Compression）
**对话太长就自动"做笔记"**

**优化策略**:
- 丢掉细节，只留要点
- 用更少脑容量记住最重要信息
- 快要触顶时自动压缩

#### 3️⃣ 临时工只留结论
**子 Agent 用完即毁（为了不撑爆记忆）**

**优化策略**:
- 思考过程、中间步骤、纠结——全部丢弃
- 只保留最终结果
- 节省主 Agent 的记忆空间

### 第四原理
**Agent 的瓶颈不是智商，是记忆**
