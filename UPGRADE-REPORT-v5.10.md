**文件**: `.learnings/design-patterns/api-pagination-pattern.md`

  - while 循环 + has_more
  - for 循环 + 最大页数限制
  - 递归（不推荐）
- ✅ 防御性编程技巧
  - 数据验证
  - 日志记录
  - 进度报告
- ✅ 适用场景分析
- ✅ 最佳实践（DO & DON'T）

**代码示例**:
```python
def get_all_records(api, page_size=100):
    all_records = []
    page_token = None

    while True:
        response = api.list_records(
            page_size=page_size,
            page_token=page_token
        )
        all_records.extend(response['records'])

        if not response.get('has_more'):
            break

        page_token = response.get('page_token')

    return all_records
```

## 🐛 错误修复

### 错误 1: 飞书数据读取不完整

**问题描述**:
- 只读取了 10 条记录
- 实际有 40 条记录
- 错误率：75% 数据遗漏

**根本原因**:
- 没有处理 API 分页
- 没有检查 `has_more` 字段
- 没有验证数据总数

**修复措施**:
1. 创建 API 分页处理模式
2. 创建数据获取检查清单
3. 整合到 SOUL.md
4. 深度学习和改进

**验证结果**:
- ✅ 成功读取所有 40 条记录
- ✅ 向用户报告准确数据
- ✅ 不会再发生类似错误

### 错误 2: API Key 泄露

**问题描述**:
- `shield-guard-coordinator.sh` 包含明文 API Keys
- GitHub Secret Scanning 检测到泄露
- 推送被阻止

**修复措施**:
- 移除明文密钥
- 改用环境变量读取
- 添加环境变量检查

**结果**:
- ✅ 成功推送到 GitHub
- ⚠️ 历史提交仍包含密钥（需要轮换）
