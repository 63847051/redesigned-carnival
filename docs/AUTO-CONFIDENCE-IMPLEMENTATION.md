## 🎯 功能概述

**自动置信度计算**：基于历史对话自动计算 Retain 条目（O 类型）的置信度，避免手动标注。

---

## 📊 置信度计算算法

### 算法逻辑

```
输入: O 类型条目 + 历史对话
  ↓
Step 1: 统计出现频率
  ↓
Step 2: 检查证据支持
  ↓
Step 3: 检查表述确定性
  ↓
输出: 置信度 (0.5 - 1.0)
```

### 详细规则

**规则 1: 出现频率**
- 出现 ≥ 3 次 → 置信度 0.9
- 出现 1-2 次 → 继续检查

**规则 2: 证据支持**
- 有"测试通过"、"验证成功"等证据 → 0.8
- 无证据 → 继续检查

**规则 3: 表述确定性**
- 高确定性（确定、肯定、必须） → 0.8
- 中等确定性（应该、建议） → 0.7
- 低确定性（可能、也许） → 0.6

**规则 4: 默认值**
- 无明确信号 → 0.7（中等置信度）

---

## 🔧 实现方式

### 方式 1: 手动计算（当前可用）

**步骤**:
1. 检查今日日志中的 O 类型条目
2. 手动搜索历史中是否出现过类似内容
3. 根据规则判断置信度
4. 更新条目

**示例**:
```bash
# 查看今日 O 条目
grep "^- O" memory/2026-04-02.md

# 搜索历史
grep -r "先画拓扑图" memory/*.md

# 如果出现 3+ 次，更新为 c=0.9
```

---

### 方式 2: 自动计算（开发中）

**脚本**: `scripts/auto-confidence.py`

**功能**:
- 自动扫描历史日志
- 统计关键词出现频率
- 检测证据支持
- 计算置信度
- 更新 Retain 条目

**使用方法**:
```bash
# 分析今日日志
python3 scripts/auto-confidence.py memory/2026-04-02.md

# 更新置信度
python3 scripts/auto-confidence.py --update memory/2026-04-02.md
```
