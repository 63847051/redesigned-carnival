### 2️⃣ 实现四种封闭类型

**目标**：按照 Claude Code 的标准重新组织记忆

**四种类型**：
1. **user** - 用户画像
2. **feedback** - 正负反馈（都要记录！）
3. **project** - 项目上下文
4. **reference** - 外部指针

**示例格式**：
```markdown
---
name: 用户偏好
description: terse 响应，不要总结
type: feedback
---

用户偏好 terse 响应，不需要在每次响应后总结。

**Why:** 用户明确表示"stop summarizing what you just did at the end of every response, I can read the diff"

**How to apply:** 所有回复都保持简洁，避免冗长的总结部分
```

### 3️⃣ 优化记忆召回策略

**当前配置**：
```json
"recall": {
  "enabled": true,
  "strategy": "keyword",
  "maxResults": 5,
  "scoreThreshold": 0.3
}
```

**理解**：
- ✅ keyword 模式：关键词搜索（不需要 embedding）
- ✅ 最多返回 5 篇
- ✅ 相关性阈值 0.3

**为什么这样设计？**
- Claude Code 用 Sonnet 做智能筛选
- 我们用 keyword 模式（更轻量）
- 不确定的就不选（避免污染）

---

### 4️⃣ 实现双轨注入

**目标**：理解并应用双轨注入

**双轨**：
- **通道A**：指令记忆 → UserMessage
- **通道B**：行为规范 → System Prompt

**应用**：
- 指令记忆多变，需要频繁更新
- 行为规范稳定，整个会话只计算一次
- 缓存策略独立管理

---

### 5️⃣ 实现两回合策略

**目标**：优化自动记忆提取

**两回合**：
- **第一回合**：并行读取所有可能需要的文件
- **第二回合**：并行写入所有需要更新的文件

**为什么？**
- FILE_EDIT_TOOL_NAME 要求先 FILE_READ_TOOL_NAME
- 最大化并行效率
- 减少轮次消耗
