## 🎯 核心理念

**Retain 格式**是一种结构化记忆提取方法，用于从每日对话中提取关键信息，形成可检索、可复用的知识库。

**核心原则**:
- 📝 **结构化**: 使用固定格式，便于机器解析
- 🏷️ **分类标签**: W/B/O 三种类型，清晰区分
- 🎯 **带上下文**: @标签 标记信息来源和领域
- 📊 **信心度**: 观点类信息标注置信度

---

## 📋 三种标签类型

### 1. W - World Facts（世界事实）

**定义**: 客观的、持久的事实性知识

**特征**:
- ✅ 不依赖主观判断
- ✅ 长期有效
- ✅ 可验证

**示例**:
```markdown
- W @飞书API: 飞书不支持 img 标签，必须用 upload_image API
- W @OpenClaw: OpenClaw 的配置文件位于 /root/.openclaw/openclaw.json
- W @服务器: 主服务器 IP 是 43.134.63.176
```

**适用场景**:
- API 规范和限制
- 系统配置和路径
- 技术事实和约束
- 平台特性

---

### 2. B - Behavior（行为记录）

**定义**: Agent 执行的具体行动和项目进展

**特征**:
- ✅ 记录已经发生的事情
- ✅ 可追溯的历史
- ✅ 项目里程碑

**示例**:
```markdown
- B @CEO-Agent: 今日完成选题审核，发布3篇小红书
- B @项目: 完成 Agent-Reach 安装，9/16 渠道可用
- B @系统: 更新 MEMORY.md，压缩至 2833 字符
```

**适用场景**:
- 任务完成情况
- 项目里程碑
- 系统更新记录
- 决策执行状态

---

### 3. O - Opinions（观点和偏好）

**定义**: 带主观判断的观点、偏好、趋势观察

**特征**:
- ✅ 主观判断
- ✅ 带不确定性
- ✅ 需要标注信心度

**格式**: `O(c=信心度) @领域: 观点内容`

**示例**:
```markdown
- O(c=0.9) @内容策略: 深度长文比短平快内容更受目标读者欢迎
- O(c=0.7) @技术: 向量搜索比 grep 更适合大规模日志检索
- O(c=0.5) @趋势: AI Agent 记忆系统可能会向语义检索方向发展
```

**信心度指南**:
- `c=1.0`: 完全确定（几乎不会错）
- `c=0.8-0.9`: 高度确定（有证据支持）
- `c=0.6-0.7`: 中等确定（部分证据）
- `c=0.4-0.5`: 低确定性（推测为主）
- `c=0.2-0.3`: 很不确定（仅作参考）

**适用场景**:
- 策略和方向判断
- 趋势观察
- 用户偏好
- 技术选择评估

## 📝 每日日志模板

在每日日志文件 `memory/YYYY-MM-DD.md` 中添加以下结构：

```markdown
# YYYY-MM-DD 日志

## Retain

### 世界知识 (W)
- W @领域: 具体事实

### 行为记录 (B)
- B @Agent/项目: 完成的工作

### 观点偏好 (O)
- O(c=0.9) @领域: 带信心度的观点

## 今日工作详情
[正常日志内容...]

## 关键决策
[决策记录...]

## 问题与解决
[问题记录...]
```

## 🔍 提取指南

### 何时提取？

**触发条件**:
- 对话包含纠正反馈
- 做出重要决策
- 学习到新的 API/平台特性
- 用户表达明确偏好
- 项目完成里程碑
- 系统配置变更

### 提取原则

1. **最小化**: 只提取真正需要长期保留的信息
2. **精确性**: 使用具体数值和明确描述
3. **可检索**: 使用 @标签 标记领域和上下文
4. **非冗余**: 不记录可推导的信息

---

## 🚫 不适合提取的内容

### ❌ 不要提取：

- 临时性调试信息
- 会话状态信息（已在 SESSION-STATE.md）
- 可从代码推导的事实
- 短期任务细节
- 重复信息

### ✅ 应该提取：

- 跨会话有效的规则
- 重要的错误教训
- 用户明确偏好
- 系统约束和限制
- 项目里程碑
- 技术决策原因

## 📊 使用场景

### 场景 1: API 学习

**对话**:
> "飞书不支持 HTML 的 img 标签，必须使用 upload_image API"

**提取**:
```markdown
- W @飞书API: 飞书不支持 img 标签，必须用 upload_image API
```

---

### 场景 2: 用户偏好

**对话**:
> "我喜欢简洁的回复，不要啰嗦"

**提取**:
```markdown
- O(c=0.9) @沟通: 用户喜欢简洁的回复风格，避免冗长解释
```

---

### 场景 3: 项目里程碑

**对话**:
> "Agent-Reach 安装完成，9/16 渠道可用"

**提取**:
```markdown
- B @项目: 完成 Agent-Reach 安装，9/16 渠道可用
```

---

### 场景 4: 技术决策

**对话**:
> "我们选择使用智谱 AI embeddings，因为免费且 1024 维"

**提取**:
```markdown
- O(c=0.8) @技术: 选择智谱 AI embeddings（免费、1024维）用于记忆搜索
```

## 📈 效果评估

### 成功指标

1. **检索效率**: 能通过标签快速找到相关信息
2. **信息密度**: 每日日志的 Retain 段落控制在 10-20 条
3. **复用率**: 历史信息被正确引用和使用
4. **准确性**: 记录的事实和观点长期有效
