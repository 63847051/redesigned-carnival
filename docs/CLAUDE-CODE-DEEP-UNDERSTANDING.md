## 💡 核心设计理解

### 1. 双轨注入 ⭐⭐⭐⭐⭐

**核心洞察**：
```typescript
// 通道A: 指令记忆 → UserMessage（systemReminder 标签）
// 通道B: 行为规范 → System Prompt 数组

// 为什么分离？
// - 指令记忆多变，需要频繁更新
// - 行为规范稳定，整个会话只计算一次
// - 缓存策略独立管理
```

**关键价值**：
- ✅ 指令记忆和行为规范分离
- ✅ 缓存策略独立管理
- ✅ token 预算灵活分配
- ✅ 通道B整个会话只计算一次

---

### 2. 四种封闭类型 ⭐⭐⭐⭐⭐

**核心洞察**：
```typescript
// user: 用户画像（个性化协作）
// feedback: 正负反馈都要记录（避免过度保守）
// project: 项目上下文（不可从代码推导）
// reference: 外部指针（外部资源）

// 什么不存？
// - 代码模式、架构、文件路径、git 历史
// - 原因：代码就是最权威的来源
```

**反直觉设计**：
- ✅ **feedback: 正负反馈都要记录**
- ❌ 只记录错误 → AI 会变得过度保守
- ✅ 记录成功 → AI 知道哪些做法是对的

**关键价值**：
- ✅ 高信噪比（只存储不可推导的信息）
- ✅ 避免变成万能笔记本
- ✅ 保持记忆系统有效性

---

### 3. Sonnet 动态召回 ⭐⭐⭐⭐⭐

**核心洞察**：
```
不是向量检索，是 AI 决策任务！
- 异步预取，不阻塞主流程
- 最多返回 5 篇
- 不确定的就不选
```

**关键约束**：
- ✅ 已经展示过的过滤掉
- ✅ 工具文档压低优先级
- ✅ 不确定的就不选

**关键价值**：
- ✅ 质量优于数量
- ✅ 控制成本（轻量级模型）
- ✅ 不阻塞主流程

---

### 4. API 不变量保护 ⭐⭐⭐⭐⭐

**核心洞察**：
```typescript
// tool call 对完整性
// thinking block 关联性
// 压缩边界不能切断
```

**关键约束**：
- ✅ 每个 tool_use 必须有对应的 result
- ✅ 压缩边界不能切断 tool call 对
- ✅ thinking block 关联性不能破坏

**关键价值**：
- ✅ 维护 API 协议完整性
- ✅ 避免 API 报错
- ✅ 确保消息完整性

---

### 5. 三层缓存 ⭐⭐⭐⭐⭐

**核心洞察**：
```typescript
// L1: getMemoryFiles 缓存（解析结果）
// L2: getUserContext 缓存（拼接后对象）
// L3: systemPromptSection 缓存（按 section 名）

// 关键事实：没有对 CLAUDE.md 的热监听！
// 宁可牺牲实时性，也要保证确定性
```

**关键事实**：
- ✅ 没有对 CLAUDE.md 的热监听
- ✅ 宁可牺牲实时性，也要保证确定性
- ✅ 缓存一致性优先

**关键价值**：
- ✅ 行为可预测
- ✅ 避免复杂的一致性问题
- ✅ 简化实现
