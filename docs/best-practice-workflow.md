## 🎯 **核心工作流**

### **5 步循环**
```
Research → Plan → Execute → Review → Ship
```

---

## 📋 **详细步骤**

### 1️⃣ **Research（研究）**
**目标**: 理解问题和需求

**活动**：
- ✅ 收集信息
- ✅ 分析需求
- ✅ 研究技术方案
- ✅ 评估可行性

**输出**：
- 需求文档
- 技术调研报告
- 可行性分析

---

### 2️⃣ **Plan（规划）**
**目标**：制定执行计划

**活动**：
- ✅ 任务分解
- � 优先级排序
- ✅ 资源评估
- ✅ 时间估算

**输出**：
- 任务列表
- 时间表
- 资源分配

---

### 3️⃣ **Execute（执行）**
**目标**：按计划执行

**活动**：
- ✅ 编写代码
- ✅ 实现功能
- ✅ 单元测试
- ✅ 集成测试

**输出**：
- 源代码
- 测试报告
- 部署文档

---

### 4️⃣ **Review（审查）**
**目标**：质量检查

**活动**：
- ✅ 代码审查
- ✅ 安全审查
- ✅ 性能测试
- ✅ 用户测试

**输出**：
- 审查报告
- 问题列表
- 改进建议

---

### 5️⃣ **Ship（发布）**
**目标**：发布上线

**活动**：
- ✅ 修复问题
- � 最终测试
- ✅ 部署上线
- ✅ 发布通知

**输出**：
- 发布版本
- 发布说明
- 用户文档

## 🎯 **Command → Agent → Skill 模式**

### **工作流**
```
用户 → Command → Agent → Skill → 输出
```

### **示例**

#### 场景 1: 代码审查
```
用户: /code-review
Command: code-review 命令
Agent: security-auditor subagent
Skill: security-check 技能
输出: 审查报告
```
