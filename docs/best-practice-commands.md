## 🎯 **Commands 最佳实践**

### 📋 **什么是 Command？**

Command 是**注入到现有上下文的提示词模板**：
- ✅ 简单的用户调用
- ✅ 工作流编排
- ✅ 可复用的流程

### 🎯 **何时使用 Command？**

**适用场景**：
- ✅ **工作流编排** - 多步骤任务
- ✅ **频繁操作** - 重复性工作
- ✅ **快捷方式** - 简化复杂操作

**不适用场景**：
- ❌ 一次性任务
- ❌ 简单查询
- ❌ 动态内容

---

## 🚀 **创建 Command**

### 方法 1: 通过文件创建

**位置**: `.claude/commands/<name>.md`

**模板**：
```markdown
# <Command Name>

## 触发词
<用户如何触发>

## 描述
<命令的简短描述>

## 步骤
1. 步骤 1
2. 步骤 2
3. 步骤 3

## 提示词
<具体的提示词内容>

## 输出格式
<期望的输出格式>
```

### 方法 2: 通过对话创建

**在 Claude Code 中**：
```
> 创建一个命令，用于 <task>
```

**Claude 会自动**：
1. 创建命令文件
2. 设置触发词
3. 测试命令

## 📊 **示例 Commands**

### 1️⣣ **代码审查命令**
```markdown
# Code Review

## 触发词
/code-review

## 描述
快速审查代码质量和安全问题

## 步骤
1. 分析代码结构
2. 检查安全问题
3. 生成审查报告

## 提示词
请审查以下代码：
- 代码质量
- 安全漏洞
- 性能问题
- 最佳实践

## 输出格式
## 审查报告

### 代码质量
- 评分：/10
- 问题：...

### 安全
- 风险等级：...
- 漏洞：...

### 性能
- 瓶颈：...
- 建议：...
```

### 2️⣣ **文档生成命令**
```markdown
# Generate Docs

## 触发词
/gen-docs

## 描述
从代码生成文档

## 步骤
1. 分析代码结构
2. 提取关键信息
3. 生成 Markdown 文档

## 提示词
请为以下代码生成文档：
- API 文档
- 使用示例
- 架构说明

## 输出格式
```markdown
# API 文档

## 概述
...

## 接口
...

## 示例
...
```
```
