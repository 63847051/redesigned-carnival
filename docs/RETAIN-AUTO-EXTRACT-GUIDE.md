# Retain 自动提取工具使用指南

**创建时间**: 2026-04-02 16:50
**版本**: v1.0

---

## 🎯 工具概述

**Retain 自动提取工具**可以从对话内容中自动识别并提取 W/B/O 三种类型的结构化记忆条目。

**两个脚本**:
- `extract-retain.py` - Python 核心引擎
- `auto-retain.sh` - Shell 便捷包装

---

## 📋 使用方法

### 方法 1: 直接提供文本（推荐）

```bash
bash scripts/auto-retain.sh "飞书不支持 img 标签，必须使用 upload_image API"
```

**优点**: 简单快捷
**适用**: 短到中等长度的文本

---

### 方法 2: 从标准输入

```bash
echo "完成了记忆清理脚本" | bash scripts/auto-retain.sh -
```

**优点**: 可以管道操作
**适用**: 脚本集成

---

### 方法 3: 从文件读取

```bash
bash scripts/auto-retain.sh conversation.txt
```

**优点**: 处理长文本
**适用**: 已保存的对话记录

---

### 方法 4: 仅输出，不追加

```bash
bash scripts/auto-retain.sh -n "完成了任务"
```

**优点**: 输出到终端，可手动编辑
**适用**: 需要手动调整的场景

---

## 🎯 提取效果

### ✅ 提取效果好的场景

**1. 技术规范和限制**
```
输入: "飞书不支持 HTML 的 img 标签，必须使用 upload_image API"
输出: W @API: 飞书不支持 HTML 的 img 标签，必须使用 upload_image API
```

**2. 任务完成**
```
输入: "我们完成了记忆清理脚本的创建"
输出: B @记忆系统: 我们完成了记忆清理脚本的创建
```

**3. 观点和建议**
```
输入: "我认为好的记忆系统不是存得更多，而是忘得更聪明"
输出: O(c=0.7) @记忆系统: 我认为好的记忆系统不是存得更多，而是忘得更聪明
```

---

### ⚠️ 提取效果一般的场景

**1. 高度技术性内容**
```
输入: "OpenCode CLI 是独立系统，有自己的配置"
输出: 可能无法提取或分类不准确
建议: 手动提取
```

**2. 复杂的多层信息**
```
输入: 包含多个不同主题的长段落
输出: 可能只提取部分信息
建议: 拆分为多个短段落分别提取
```

**3. 隐含信息**
```
输入: "和上次一样的问题"
输出: 无法提取（缺少上下文）
建议: 明确描述具体问题
```

## 💡 最佳实践

### 1. 对话结束时使用

**流程**:
1. 回顾今天的对话
2. 总结 3-5 个关键点
3. 运行自动提取
4. 检查并调整结果

**示例**:
```bash
# 提取今天的要点
bash scripts/auto-retain.sh "今天完成了记忆系统改进，创建了 8 个工具脚本，学习了 Retain 格式"
```

---

### 2. 手动优化提取结果

**步骤**:
1. 使用 `-n` 参数仅输出
2. 复制输出结果
3. 手动调整分类和内容
4. 追加到今日日志

**示例**:
```bash
# 仅输出
bash scripts/auto-retain.sh -n "关键内容"

# 手动编辑后追加
# 编辑输出，然后运行：
bash scripts/wal-update-memory.sh retain W "领域" "优化后的内容"
```

---

### 3. 分批提取长对话

**方法**: 将长对话拆分为多个主题

```bash
# 主题 1: 技术学习
bash scripts/auto-retain.sh "学习了 Claude Code 记忆系统"

# 主题 2: 工作完成
bash scripts/auto-retain.sh "完成了 WAL 验证工具创建"

# 主题 3: 经验总结
bash scripts/auto-retain.sh "Wesley 的实战经验非常有价值"
```

## 🔧 提取逻辑

### W（世界知识）识别

**关键词**:
- 不支持、必须使用、不能用、需要用
- API、接口、平台、系统
- 配置、服务器、IP、路径

**示例**:
- "飞书不支持 img 标签" → W @API
- "服务器 IP 是 43.134.63.176" → W @系统

---

### B（行为记录）识别

**关键词**:
- 完成、创建、更新、删除、添加
- 任务、脚本、文档、工具
- 测试、验证、通过、成功

**示例**:
- "完成了脚本创建" → B @工具
- "测试通过" → B @测试

---

### O（观点偏好）识别

**关键词**:
- 认为、建议、觉得、建议
- 更好、更优、推荐
- 原则、策略、方向

**信心度**:
- "确定"、"肯定" → c=0.9
- "应该"、"建议" → c=0.8
- "可能"、"也许" → c=0.6

**示例**:
- "建议使用智谱 AI" → O(c=0.8) @技术
- "可能需要优化" → O(c=0.6) @优化

## 📊 提取统计

运行后会显示提取统计：
```
[✓] 提取完成: W=1, B=2, O=1
```

**含义**:
- W=1: 提取了 1 条世界知识
- B=2: 提取了 2 条行为记录
- O=1: 提取了 1 条观点偏好

## ⚡ 性能优化

### 1. 去重机制

自动检测相似内容（60% 阈值），保留得分更高的版本。

### 2. 智能分类

基于关键词权重和模式匹配，选择最可能的分类。

### 3. 域提取

自动识别相关领域（API、系统、工具等）。

---

## 🚨 注意事项

1. **不是 100% 准确**
   - 自动提取是辅助工具
   - 需要人工检查和调整

2. **适用常见场景**
   - 日常对话记录
   - 任务完成汇报
   - 知识学习总结

3. **技术内容需手动**
   - 复杂的技术细节
   - 需要上下文的信息
   - 隐含的知识

## 🎓 学习建议

**初期（第 1 周）**:
- 每次对话后使用自动提取
- 检查提取结果
- 手动调整优化

**中期（第 2-4 周）**:
- 总结哪些内容提取效果好
- 调整输入文本格式
- 提高提取准确性

**后期（1 个月后）**:
- 形成固定的记忆记录习惯
- 结合自动和手动提取
- 持续优化提取质量
