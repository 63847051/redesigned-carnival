## 🚨 之前的失败方案（v2.0）

### 方案描述
直接修改 `openclaw.json` 配置文件，添加 `sessions.spawn` 配置。

### 问题
- ❌ `sessions` 键在 OpenClaw 2026.2.26 版本中**不存在**
- ❌ Gateway 检测到无效配置，运行 `doctor` 修复
- ❌ 删除 `sessions` 键后触发重启
- ❌ 如果脚本还在运行，会重新添加 `sessions` 键
- ❌ **导致无限循环重启**（15+ 次）

### 失败原因
1. **版本不兼容** - 没有验证当前版本是否支持该配置
2. **直接修改生产配置** - 没有先在测试环境验证
3. **无限循环** - 自动化脚本没有失败处理

### 影响
- Gateway 崩溃 15+ 次
- 持续时间 4 分钟
- 需要手动干预停止脚本

## 🚀 安全方案详解

### 方案 A: 环境变量方案

#### 原理
通过 systemd 服务的 `EnvironmentFile` 指令加载环境变量，**不修改配置文件**。

#### 实施步骤
1. 创建环境变量文件 `/root/.openclaw/gateway.env`
2. 修改 systemd 服务文件，添加 `EnvironmentFile=/root/.openclaw/gateway.env`
3. 重新加载 systemd 并重启 Gateway
4. 验证 `agents_list` 显示 `allowAny: true`

#### 优点
- ✅ 不修改配置文件
- ✅ 不触发配置验证
- ✅ 易于回滚（删除环境变量文件即可）
- ✅ 符合 systemd 最佳实践

#### 缺点
- ⚠️ 需要重启 Gateway
- ⚠️ 需要修改服务文件

#### 成功率
**85%** - 如果环境变量被正确支持

---

### 方案 B: ACP 模式

#### 原理
使用 `sessions_spawn` 的 `runtime: "acp"` 模式，**不需要任何配置**。

#### 实施步骤
1. 直接在主控 Agent 中运行：
```javascript
sessions_spawn({
  runtime: "acp",
  mode: "session",
  thread: true,
  model: "glmcode/glm-4.7",
  label: "设计专家",
  task: "..."
})
```

#### 优点
- ✅ **完全不需要配置**
- ✅ 不需要重启 Gateway
- ✅ 立即可用

#### 缺点
- ⚠️ 可能不支持所有功能
- ⚠️ 需要验证是否支持独立 Agent

#### 成功率
**60%** - 取决于 ACP 模式的支持程度

---

### 方案 C: 混合方案（最推荐）⭐

#### 原理
先尝试方案 B（ACP 模式），如果失败再使用方案 A（环境变量）。

#### 实施步骤
1. **尝试 ACP 模式**（1 分钟）
   - 运行测试命令
   - 如果成功 → 完成！
   - 如果失败 → 继续

2. **使用环境变量**（5 分钟）
   - 创建环境变量文件
   - 修改服务文件
   - 重启 Gateway
   - 验证结果

3. **如果都失败**
   - 回滚所有更改
   - 保持当前系统

#### 优点
- ✅ **最安全** - 两种方式都尝试
- ✅ **最灵活** - 适应不同情况
- ✅ **成功率最高** - 如果 ACP 不行，用环境变量
- ✅ **易回滚** - 1 分钟恢复

#### 成功率
**95%** - 几乎肯定成功
