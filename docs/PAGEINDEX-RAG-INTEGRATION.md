## 🎯 核心思路

将 **pageindex-rag** 的**无向量检索**思路整合到你的记忆系统中：

### **传统方法 vs pageindex-rag**

| 特性 | 传统 RAG | pageindex-rag | 你的系统 |
|------|---------|--------------|---------|
| **检索方法** | 向量嵌入 | LLM 推理 | QMD + Embeddings |
| **存储** | 向量数据库 | JSON | 本地文件 |
| **排序** | 余弦相似度 | LLM 推理 | 分数排序 |

---

## 🚀 集成方案

### **方案 1: 混合检索（推荐）** ⭐

**核心思路**: 快速召回 + LLM 排序

```
用户查询
    ↓
Step 1: 快速召回（QMD）
    ↓ 召回 10 条
Step 2: LLM 推理排序
    ↓ 选择 Top 3
返回结果
```

**优势**：
- ✅ **快速** - QMD 毫秒级召回
- ✅ **准确** - LLM 理解语义
- ✅ **简单** - 无需向量数据库

**代码**: `/root/.openclaw/workspace/scripts/hybrid-retriever-pageindex-style.py`

---

### **方案 2: 纯推理检索**

**核心思路**: 完全使用 LLM 检索

```
用户查询
    ↓
扫描所有文件名
    ↓ 构建候选列表
LLM 推理选择
    ↓
返回 Top 3
```

**优势**：
- ✅ **最简单** - 无需额外工具
- ✅ **最智能** - LLM 完全理解
- ⚠️ **较慢** - 需要 LLM 调用

**适用场景**：
- 文档数量 < 100
- 对速度要求不高
- 需要深度理解

---

### **方案 3: 分层检索**

**核心思路**: 按文档类型分层

```
用户查询
    ↓
分类：系统/用户/项目
    ↓
每类 Top 3
    ↓
LLM 融合
返回结果
```

**优势**：
- ✅ **结构化** - 按类型组织
- ✅ **全面** - 覆盖所有类型
- ✅ **灵活** - 可调整权重

## 🔧 实现细节

### **混合检索实现**

```python
class HybridRetriever:
    def retrieve(self, query: str, top_k: int = 10) -> List[Dict]:
        # Step 1: 快速召回
        initial_results = self._quick_recall(query, top_k)
        
        # Step 2: LLM 排序
        ranked_results = self._llm_rerank(query, initial_results)
        
        # Step 3: 返回 Top 3
        return ranked_results[:3]
```

### **LLM 排序提示词**

```python
prompt = f"""你是一个智能检索助手。请根据用户查询，对以下候选文档进行排序。

用户查询: {query}

候选文档:
[0] {path1}
[1] {path2}
...

任务:
1. 分析每个文档与查询的相关性
2. 按相关性从高到低排序
3. 只返回排序后的索引号，用逗号分隔

示例格式: 3,1,5,2,4

现在开始排序:"""
```

### **缓存策略**

```python
# 缓存 LLM 排序结果
cache = {}

def retrieve_with_cache(query):
    if query in cache:
        return cache[query]
    
    results = retrieve(query)
    cache[query] = results
    
    # 5分钟后过期
    setTimeout(() => del cache[query], 300000)
    
    return results
```
