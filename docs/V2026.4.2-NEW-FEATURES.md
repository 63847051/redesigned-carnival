## 1️⃣ Task Flow 系统 🎯

### 原理

**Task Flow 是什么？**
- 持久化的后台任务编排系统
- 可以独立于 Gateway 运行
- 支持任务状态跟踪和恢复

**核心特性**:
- ✅ **托管 vs 镜像同步模式**: 托管模式由 Gateway 管理，镜像模式由外部编排器控制
- ✅ **持久化流状态**: 任务状态持久化，Gateway 重启不丢失
- ✅ **修订跟踪**: 记录任务的所有变更
- ✅ **检查和恢复原语**: `openclaw flows` 命令

### 工作流程

```
创建任务流 → 执行任务 → 持久化状态 → 可恢复 → 完成
```

### CLI 命令

```bash
# 列出所有任务流
openclaw flows list

# 查看任务流详情
openclaw flows info <flow-id>

# 恢复失败的任务流
openclaw flows recover <flow-id>

# 取消任务流
openclaw flows cancel <flow-id>

# 查看任务流状态
openclaw flows status
```

### 配置示例

```yaml
# openclaw.json
agents:
  defaults:
    taskFlow:
      enabled: true
      mode: "managed"  # managed | mirrored
      persistence:
        enabled: true
        path: "/root/.openclaw/flows"
```

### 使用场景

**场景 1: 长时间运行的任务**
```bash
# 创建一个持续监控的任务流
openclaw flows create \
  --name "监控网站状态" \
  --every "5m" \
  --message "检查 https://example.com 状态"
```

**场景 2: 多步骤工作流**
```bash
# 创建一个多步骤的数据处理任务流
openclaw flows create \
  --name "数据处理流水线" \
  --steps "fetch,process,save" \
  --message "执行数据处理流水线"
```

**场景 3: 恢复失败的任务**
```bash
# 查看失败的任务流
openclaw flows list --status failed

# 恢复任务流
openclaw flows recover <flow-id>
```

### 高级特性

**托管子任务生成**:
- 父任务可以生成子任务
- 支持粘性取消意图（sticky cancel intent）
- 外部编排器停止调度后，父任务等待子任务完成

**API 运行时缝**:
- `api.runtime.taskFlow` 边界
- 插件可以创建和管理托管的 Task Flows
- 无需每次传递所有者标识符

### 实战示例

**示例 1: 定时数据采集**
```bash
# 创建任务流
openclaw flows create \
  --name "每小时采集数据" \
  --cron "0 * * * *" \
  --message "采集数据并保存到数据库"

# 查看状态
openclaw flows status

# 查看历史
openclaw flows history
```

**示例 2: 失败恢复**
```bash
# 列出失败的任务流
openclaw flows list --status failed

# 恢复特定任务流
openclaw flows recover <flow-id>

# 重新运行
openclaw flows rerun <flow-id>
```

## 2️⃣ before_agent_reply 钩子 🪝

### 原理

**before_agent_reply 钩子是什么？**
- 插件可以在 Agent 回复之前拦截
- 可以用合成回复短路 LLM
- 支持内联操作后的快速响应

**工作流程**:
```
用户消息 → 插件拦截 → before_agent_reply → 合成回复 → 跳过 LLM → 直接返回
```

### 使用场景

**场景 1: 快速命令响应**
```javascript
// 插件代码
async function before_agent_reply(context) {
  const message = context.message;

  // 检查是否是快速命令
  if (message === "/status") {
    return {
      reply: "✅ 系统运行正常",
      skip_llm: true  // 跳过 LLM
    };
  }
}
```

**场景 2: 缓存响应**
```javascript
async function before_agent_reply(context) {
  const cacheKey = generateCacheKey(context.message);

  // 检查缓存
  const cached = await cache.get(cacheKey);
  if (cached) {
    return {
      reply: cached,
      skip_llm: true
    };
  }
}
```

**场景 3: 内联操作后的确认**
```javascript
async function before_agent_reply(context) {
  // 检查是否刚刚执行了内联操作
  if (context.lastAction === "file_created") {
    return {
      reply: "✅ 文件已创建",
      skip_llm: true
    };
  }
}
```

### 配置示例

```yaml
# openclaw.json
plugins:
  entries:
    my-plugin:
      hooks:
        before_agent_reply:
          enabled: true
          priority: 100
```

### CLI 命令

```bash
# 查看所有钩子
openclaw hooks list

# 启用 before_agent_reply
openclaw hooks enable before_agent_reply

# 查看钩子详情
openclaw hooks info before_agent_reply

# 测试钩子
openclaw hooks test before_agent_reply
```

### 高级特性

**优先级控制**:
- 钩子可以设置优先级
- 高优先级先执行
- 任何钩子返回 `skip_llm: true` 都会短路

**上下文访问**:
- 可以访问完整的消息上下文
- 可以访问会话状态
- 可以访问 Agent 配置

## 3️⃣ 高级 Browser 自动化 🌐

### 原理

**Browser Control 的高级特性**:
- CDP (Chrome DevTools Protocol) 深度集成
- 支持复杂的多步骤操作
- 支持等待和条件执行
- 支持自定义脚本

### 高级命令

**1. 等待操作**
```bash
# 等待元素出现
openclaw browser wait --selector ".loaded" --timeout 10000

# 等待导航完成
openclaw browser wait --navigation

# 等待 JavaScript 条件
openclaw browser wait --js "document.readyState === 'complete'"
```

**2. 复杂交互**
```bash
# 拖放操作
openclaw browser drag \
  --start-ref "#draggable" \
  --end-ref "#dropzone"

# 选择下拉选项
openclaw browser select \
  --ref "select[name='country']" \
  --value "CN"

# 滚动到元素
openclaw browser scroll \
  --ref ".footer" \
  --align "bottom"
```

**3. JavaScript 执行**
```bash
# 执行脚本
openclaw browser eval \
  --js "document.title = 'New Title'"

# 执行脚本并获取结果
openclaw browser eval \
  --js "document.querySelector('.price').textContent" \
  --output

# 异步脚本
openclaw browser eval \
  --js "await fetch('/api/data').then(r => r.json())"
```

**4. 高级截图**
```bash
# 全页截图
openclaw browser screenshot --full-page

# 元素截图
openclaw browser screenshot \
  --selector ".dashboard" \
  --output "dashboard.png"

# 高 DPI 截图
openclaw browser screenshot \
  --dpr 2.0 \
  --quality 95
```

### 实战示例

**示例 1: 自动化登录流程**
```bash
# 导航到登录页
openclaw browser navigate https://example.com/login

# 等待表单加载
openclaw browser wait --selector "#login-form"

# 填写表单
openclaw browser type --ref "#email" --text "user@example.com"
openclaw browser type --ref "#password" --text "password"

# 点击登录
openclaw browser click --ref "button[type='submit']"

# 等待跳转
openclaw browser wait --navigation

# 验证登录成功
openclaw browser eval --js "document.querySelector('.user-profile')" --output
```

**示例 2: 数据抓取**
```bash
# 导航到数据页面
openclaw browser navigate https://example.com/data

# 等待数据加载
openclaw browser wait --selector ".data-table"

# 抓取数据
openclaw browser eval \
  --js "Array.from(document.querySelectorAll('.data-table tr')).map(tr => Array.from(tr.querySelectorAll('td')).map(td => td.textContent))" \
  --output > /tmp/data.json

# 截图保存
openclaw browser screenshot --full-page --output /tmp/data.png
```

**示例 3: 定期监控**
```bash
# 创建定期监控任务
openclaw cron add \
  --every "30m" \
  --name "监控网站" \
  --message "
    导航到 https://status.example.com
    等待页面加载
    截图保存
    检查是否有错误信息
    如果有错误，发送通知
  "
```

### 配置优化

**性能优化**:
```yaml
browser:
  enabled: true
  # 禁用图片加载
  args:
    - "--blink-settings=imagesEnabled=false"
  # 设置缓存大小
  args:
    - "--disk-cache-size=104857600"
```

**稳定性优化**:
```yaml
browser:
  enabled: true
  # 重试配置
  retry:
    maxAttempts: 3
    retryDelay: 1000
  # 超时配置
  timeout:
    default: 30000
    navigation: 60000
```
