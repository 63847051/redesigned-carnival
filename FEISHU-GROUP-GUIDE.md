# 📱 飞书群聊使用指南 - Multi-Agent 并行处理系统

**更新时间**: 2026-03-12
**状态**: ✅ 可用

---

## 🎯 快速开始

### 在任意飞书群聊中使用

由于你的飞书配置是 `groupPolicy: "open"`，**所有群聊都可以使用**！

---

## 📋 使用步骤

### Step 1: 确保机器人在群里

1. 打开飞书群聊
2. 添加机器人（如果还没有）
3. 确保机器人有权限接收消息

### Step 2: 使用 @ 触发

在群聊中直接输入：

```
@大领导 今天要完成的任务：
  1. @设计 修改3F更衣室排砖图
  2. @技术 写个脚本抓取数据
  3. @小蓝 更新工作日志
```

### Step 3: 等待结果

系统会自动：
1. 识别 @ 提及
2. 分配给对应专家
3. 并行执行任务
4. 汇总结果并回复

---

## 👥 可用角色

| 触发词 | 角色 | 职责 | 模型 |
|--------|------|------|------|
| `@大领导` | 任务总监 | 分配任务、监督、汇总 | GLM-4.7 |
| `@设计` | 设计专家 | 室内设计、图纸 | GLM-4.7 |
| `@技术` | 技术专家 | 编程、爬虫、API | GPT-OSS-120B（免费）|
| `@小蓝` | 工作日志 | 记录、更新、统计 | GLM-4.5-Air（免费）|

## 💡 最佳实践

### ✅ 推荐做法

1. **明确指定 @ 角色**
   ```
   好的: @设计 修改3F更衣室排砖图
   不好的: 帮我修改一下排砖图
   ```

2. **简洁描述任务**
   ```
   好的: @设计 修改3F更衣室排砖图，采用300x600瓷砖
   不好的: @设计 帮我处理一下3F那个更衣室的排砖，我觉得用300x600的瓷砖比较好，当然也可以考虑其他的...
   ```

3. **利用并行能力**
   ```
   好的: @大领导 处理这3个任务：@设计 xxx @技术 xxx @小蓝 xxx
   不好的: 一个接一个地发
   ```

4. **信任大领导**
   ```
   好的: @大领导 我有这些任务需要完成...
   不好的: 详细指定每个任务的细节
   ```

## ⚠️ 注意事项

### 当前限制

1. **任务解析**: 可能解析编号（如"2"、"3"），但不影响核心功能
2. **依赖处理**: 当前所有任务都是并行执行，不支持依赖关系
3. **响应时间**: 根据任务复杂度，通常 1-5 分钟

### 故障排查

**问题**: 机器人没有响应
- ✅ 检查机器人是否在群里
- ✅ 检查 Gateway 是否运行：`systemctl --user status openclaw-gateway`
- ✅ 检查日志：`/root/.openclaw/logs/`

**问题**: 识别错误的角色
- ✅ 使用准确的触发词（@大领导、@设计、@技术、@小蓝）
- ✅ 避免使用相似的表达
