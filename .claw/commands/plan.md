# /plan - 实现规划

## 功能
创建详细的实现计划，包括任务分解、优先级排序、资源评估。

## 使用方法
\`\`\`
/plan "实现用户认证功能"
\`\`\`

## 输入规范

### 必需参数
- **goal**: 目标描述（字符串）
  - 明确要实现什么功能
  - 示例: "实现用户认证功能"

### 可选参数
- **context**: 背景信息（字符串）
  - 当前系统状态
  - 已完成的工作
  - 相关约束条件

- **constraints**: 约束条件（数组）
  - 时间限制
  - 资源限制
  - 技术约束

- **preferences**: 偏好设置（对象）
  - 优先级策略
  - 实现方式偏好
  - 风险容忍度

## 输出规范

### 1. 任务列表（数组）
每个任务包含：
- **id**: 唯一标识符
- **title**: 任务标题
- **description**: 详细描述
- **priority**: 优先级（P0/P1/P2/P3）
- **status**: 状态（todo/in_progress/done）
- **dependencies**: 依赖任务ID列表
- **estimated_time**: 预估时间（小时）
- **assignee**: 负责Agent

### 2. 依赖关系图
- 任务之间的依赖关系
- 可视化展示（文本格式）

### 3. 风险评估
- 潜在风险列表
- 风险等级（高/中/低）
- 缓解措施

### 4. 资源需求
- 所需技能
- 所需工具
- 所需时间

## 预期输出格式

\`\`\`markdown
# 实现计划: [目标]

## 任务列表
1. [P0] 任务1 - 描述（预估: 2h）
2. [P1] 任务2 - 描述（预估: 1h）

## 依赖关系
任务1 → 任务2 → 任务3

## 风险评估
- [高] 风险1: 描述
  - 缓解: 措施

## 资源需求
- 技能: Python, Flask
- 工具: pytest, docker
- 时间: 总计 8h
\`\`\`

## 验证标准
- ✅ 所有任务都有明确描述
- ✅ 依赖关系清晰可追踪
- ✅ 优先级合理排序
- ✅ 时间估算合理
- ✅ 风险已识别

## 边界（不覆盖）
- ❌ 不包含具体的代码实现
- ❌ 不包含测试用例编写
- ❌ 不包含部署配置

## 示例

### 输入
\`\`\`
/plan "实现用户认证功能"
\`\`\`

### 输出

任务1 → 任务2 → 任务4 → 任务5
任务1 → 任务3 → 任务4 → 任务5
