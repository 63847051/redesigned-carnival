# /cost - 成本监控

## 功能
显示当前会话的 Token 使用情况和成本预估。

## 使用方法
```
/cost
```

## 输出
- 当前会话 Token 使用
- 当前会话成本
- 预估下一个操作成本
- 模型选择建议

## 示例
```
/cost

输出:
📊 当前会话统计:
  - Token 使用: 15,234 / 200,000 (7.6%)
  - 成本: ¥0.15
  - 模型: GLM-4.7
  
💡 建议:
  - 下一个简单任务可使用 GLM-4.5-Air
  - 下一个代码任务可使用 opencode/minimax-m2.5-free
```

## 成本估算

**参考定价**（估算）:
- GLM-4.5-Air: ¥0.001 / 1K tokens
- GLM-4.6: ¥0.003 / 1K tokens
- GLM-4.7: ¥0.01 / 1K tokens
- GLM-5: ¥0.05 / 1K tokens
- opencode: 免费

## 优化建议

**高成本** (使用 > 50% Token):
- 考虑使用 /compact 压缩上下文
- 切换到更便宜的模型

**中成本** (使用 20-50% Token):
- 继续使用当前模型
- 监控使用情况

**低成本** (使用 < 20% Token):
- 当前状态良好
- 无需优化
