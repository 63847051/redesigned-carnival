# 战略压缩技能

## 功能
在逻辑断点建议压缩，避免过度压缩，保持上下文质量。

## 触发时机
自动触发或手动触发：
- 上下文使用 > 50%
- 会话时间 > 30 分钟
- 完成一个里程碑后

## 何时压缩 ✅

### 推荐压缩时机
1. **研究完成后** - 开始实施前
2. **完成一个里程碑后** - 开始下一个任务前
3. **调试完成后** - 继续其他工作前
4. **失败方法尝试后** - 尝试新方法前

### 压缩策略
- **保守压缩**: 50% 使用时压缩
- **激进压缩**: 80% 使用时压缩
- **紧急压缩**: 90% 使用时压缩

## 何时不压缩 ❌

### 避免压缩的时机
1. **实施过程中** - 会丢失变量名和状态
2. **调试过程中** - 会丢失调试信息
3. **代码编写中** - 会丢失代码上下文
4. **分析过程中** - 会丢失分析结果

## 压缩方法

### 自动压缩
```json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```

### 手动压缩
```
/compact
```

## 压缩前准备

1. **保存重要信息** - 更新 SESSION-STATE.md
2. **记录当前进度** - 更新 working-buffer.md
3. **标记当前位置** - 记录任务状态

## 压缩后恢复

1. **重新加载上下文** - SessionStart Hook 自动加载
2. **恢复任务状态** - 读取 SESSION-STATE.md
3. **继续工作** - 从断点继续

## 最佳实践

### ✅ DO
- 在逻辑断点压缩
- 保存关键信息
- 使用手动压缩控制时机

### ❌ DON'T
- 在实施过程中压缩
- 在调试过程中压缩
- 过度压缩（导致信息丢失）

## 示例

### 场景 1: 研究完成后 ✅
```
研究 Golutra 项目 → /compact → 开始实施
```

### 场景 2: 实施过程中 ❌
```
编写代码中 → ❌ 不要压缩 → 完成后再压缩
```
