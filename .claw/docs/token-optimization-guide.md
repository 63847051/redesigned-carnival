## 🎛️ 上下文管理技巧

### DP-006: 子 Agent Token 优化 ⭐

**核心原则**:
1. **身份 vs 技能分离** - 子 Agent 只需要技能，不需要完整身份
2. **主从上下文分层** - 父 Agent 有完整上下文，子 Agent 只有任务上下文
3. **最小化 Token 传递** - 只传递必要的上下文

**实施**:
```json
{
  "subagents": {
    "context": {
      "excludeFiles": [
        "SOUL.md",
        "IDENTITY.md",
        "USER.md"
      ]
    },
    "timeout": 300
  }
}
```

**效果**: 65% Token 节省（子 Agent: 4300 → 1500 tokens）

### 上下文压缩

**自动压缩**:
```json
{
  "env": {
    "CLAUDE_AUTOCOMPACT_PCT_OVERRIDE": "50"
  }
}
```

**手动压缩**:
- 在逻辑断点压缩
- 研究完成后，开始实施前
- 完成一个里程碑后
- 调试完成后

### Thinking Tokens 限制

**设置**:
```json
{
  "env": {
    "MAX_THINKING_TOKENS": "10000"
  }
}
```

**效果**: 70% thinking token 减少

## 🎯 最佳实践

1. **使用免费模型** - 70% 任务使用免费模型
