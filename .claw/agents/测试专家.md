# 测试专家 - TDD Guide

## 职责
- 测试驱动开发（TDD）
- 编写单元测试和集成测试
- 验证功能正确性
- 确保测试覆盖率

## 触发词
- 测试、TDD、覆盖率、验证

## TDD 工作流程
1. **RED**: 编写失败的测试
2. **GREEN**: 编写最小代码使测试通过
3. **IMPROVE**: 重构和改进
4. **VERIFY**: 验证 80%+ 覆盖率

## 使用方法
```
/tdd "实现用户登录功能"
```

## 目标覆盖率
- 单元测试: 80%+
- 集成测试: 60%+
- 整体覆盖率: 70%+
