# 关键规则 002：版本号跟踪

**规则ID**: RULE-002
**优先级**: 🔴 CRITICAL
**创建时间**: 2026-03-13 07:38
**状态**: ✅ 激活

## 规则内容

**每次系统升级后，必须同步更新版本号**

### 触发场景
- OpenClaw 框架升级
- 核心系统文档更新（SOUL.md、IDENTITY.md、AGENTS.md 等）
- 重大功能添加或移除
- 架构调整

### 版本号存储位置
- **SOUL.md**: 主版本号（如 v5.9）
- **MEMORY.md**: 系统状态章节的版本信息
- **IDENTITY.md**: 如果包含版本信息

### 版本号格式规范
- 主版本：v5.9（大版本.小版本）
- 更新时间：YYYY-MM-DD HH:MM
- 变更说明：简要说明本次更新的核心内容

### 操作前检查清单
```
- [ ] 我是否修改了核心系统文件？
- [ ] 我是否升级了框架或依赖？
- [ ] 我是否更新了 SOUL.md 中的版本号？
- [ ] 我是否在 MEMORY.md 中记录了此次更新？
- [ ] 我是否更新了"最后升级时间"字段？
```

### 示例

**升级前**：
```markdown
## 🧬 自主进化系统 5.8
**升级时间**: 2026-03-09 12:35
```

**升级后**：
```markdown
## 🧬 自主进化系统 5.9
**升级时间**: 2026-03-13 07:38
**变更内容**: 添加 RULE-002 版本号跟踪规则
```

## 违规后果
- ❌ 版本号与实际系统状态不一致
- ❌ 无法追溯系统演进历史
- ❌ 升级回滚时定位困难
