## 🎯 需求

**你想要的**：
- 在飞书群里 @ 不同的 Agent
- 每个 Agent 处理不同类型的任务
- 并行处理，提高效率

**示例场景**：
```
用户: "@大领导 设计一个客厅"
用户: "@技术 写个 Python 脚本"
用户: "@小蓝 更新工作日志"
```

## 🚀 解决方案

### 方案：主 Agent 路由系统 ⭐ 推荐

**原理**：
- 飞书群里只有一个主 Agent（你的 bot）
- 主 Agent 监听 @ 提及
- 根据 @ 提及的文本路由到对应的子 Agent
- 并行处理任务

**架构**：
```
飞书群（快乐龙虾球）
    ↓
    @ 大领导 / @ 技术 / @ 小蓝
    ↓
主 Agent（Bot: ou_7949d5611130e7a28f84853576e9eb32）
    ↓
    ┌─────────────────────────────────────┐
    │                                     │
    ↓                                     ↓
Builder Agent                      Tech Agent
（室内设计专家）                    （技术专家）
    │                                     │
    └─────────────────────────────────────┘
                    ↓
                 Ops Agent
                （小蓝）
```

## 🎯 关键要点

### 单 Bot 模式
- 飞书群里只有 1 个 bot
- 通过 @ 提及区分不同 Agent
- 主 Agent 负责路由和协调

### 并行处理
- 多个任务可以同时处理
- 每个 Agent 独立工作
- 主 Agent 汇总结果

### 灵活扩展
- 可以轻松添加新的 Agent
- 只需添加新的触发词
- 不需要修改飞书群配置
