# 报告汇总 Skill

**Skill 名称**: `competitor-reporter`

**版本**: v1.0

---

## description

当用户要求生成竞品分析报告、汇总竞品变化、输出监控结果时使用此技能。

不适用于：非竞品数据报告、非变化数据的分析。

---

## 触发条件

- 用户要求生成报告
- 变化检测 Agent 完成后自动触发
- 用户问"生成报告"、"汇总一下"等

---

## 工作流程

### 1. 加载变化数据

- 读取 `data/diff-YYYY-MM-DD.json`
- 验证数据完整性

### 2. 生成报告摘要

- 统计总变化数
- 分类统计（新增/更新/删除）
- 生成概览段落

### 3. 趋势分析

- 计算平均内容相似度
- 识别重大更新（相似度 < 50%）
- 分析价格动态
- 分析新产品动向

### 4. 按竞品分组

- 按竞品名称分组
- 生成每个竞品的详细报告
- 包括：
  - 新增页面
  - 更新页面（标题、内容、价格）
  - 删除页面

### 5. 生成 Markdown 报告

- 标题和日期
- 变化概览
- 趋势分析
- 详细变化
- 说明部分

### 6. 保存报告

- 文件路径：`reports/YYYY-MM-DD.md`
- 格式：标准 Markdown

## 分析逻辑

### 重大更新识别

**条件**: 内容相似度 < 50%

**示例**:
- Apple 首页：33.89% → ⚠️ 重大更新
- Microsoft 首页：21.49% → ⚠️ 重大更新

### 价格变化分析

**触发**: 有价格字段变化

**建议**: "竞品正在调整价格策略，可能影响市场定位"

### 新增页面分析

**触发**: 有新增页面

**建议**: "竞品可能正在扩展产品线或增加营销页面"

## 与其他 Agent 的协作

### 上游 Agent
- **competitor-diff-detector**（变化检测 Agent）
  - 读取其输出的 diff JSON
  - 作为输入数据

### 下游 Agent
- **feishu-notifier**（推送通知 Agent）
  - 读取本 Agent 输出的 Markdown 报告
  - 推送到飞书

### 触发方式
- 上游 Agent 完成后，自动触发本 Agent
