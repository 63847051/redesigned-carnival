**版本**: v1.0

## 🎯 项目目标

创建 OpenClaw 的完整 Slock 克隆 - 本地 CLI Agent 运行时编排系统。

---

## 🏗️ 系统架构（基于 Slock）

### 核心分层

```
┌─────────────────────────────────────┐
│     Web UI (可视化界面)            │
│  - 频道/线程/任务流展示             │
│  - Agent 状态可视化                │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     Slock Server (持久化存储)       │
│  - 消息存储                         │
│  - 任务状态                         │
│  - 会话管理                         │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     Local Daemon (编排核心)         │
│  ├─ 控制平面                         │
│  ├─ 进程状态机                       │
│  ├─ 驱动适配器                       │
│  └─ 运行时 I/O                       │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     Chat Bridge MCP (协作接口)      │
│  - send_message                     │
│  - read_history                    │
│  - claim_tasks                     │
└──────────────┬──────────────────────┘
               ↓
┌─────────────────────────────────────┐
│     Runtime Entry (启动入口)        │
│  - Subagent Path                    │
│  - ACP Path                        │
│  - OpenCode Path                    │
└─────────────────────────────────────┘
```

## 📊 完整组件清单

### 1. Web UI ⭐⭐⭐⭐⭐

**功能**:
- 频道/线程/任务流展示
- Agent 状态可视化
- 实时监控
- 跨设备访问

**实现**: `web_ui.py` + `index.html`

---

### 2. Slock Server ⭐⭐⭐⭐⭐

**功能**:
- 持久化存储所有消息
- 任务状态管理
- 会话管理
- WebSocket 服务

**数据结构**:
```json
{
  "messages": [],
  "tasks": [],
  "sessions": {},
  "channels": {},
  "threads": {}
}
```

---

### 3. Local Daemon ⭐⭐⭐⭐⭐

**功能**:
- Agent 生命周期管理
- 提示词组装
- 状态缓存
- 心跳检测

**组件**:
- 控制平面（Connection、Assembly、Cache）
- 进程状态机（State Machine）
- 驱动适配器（Subagent、ACP、OpenCode）
- 运行时 I/O（Process、Parser、MCP）

---

### 4. Chat Bridge MCP ⭐⭐⭐⭐⭐

**功能**:
- 统一工具注册
- 跨 Agent 协作
- 标准化接口

**工具**:
- `send_message` - 发送消息
- `read_history` - 读取历史
- `claim_tasks` - 认领任务

---

### 5. Runtime Entry ⭐⭐⭐⭐⭐

**功能**:
- 启动本地 CLI Agent
- 管理进程生命周期
- 解析输出

**路径**:
- Subagent Path - `exec/resume`
- ACP Path - 常驻会话 + stdin
- OpenCode Path - CLI 调用

---

## 🔄 完整数据流

```
用户消息
    ↓
Local Daemon（编排）
    ↓
┌───────────┬─────────┬──────────┐
│           │         │          │
Subagent   ACP       OpenCode
Path       Path       Path
│           │         │          │
└───────────┴─────────┴──────────┘
    ↓
Slock Server（存储）
    ↓
Chat Bridge MCP（协作）
    ↓
其他 Agent
```

---

## 💡 关键差异：Codex vs Claude

| 特性 | Subagent | ACP |
|------|----------|-----|
| **核心模式** | 重启模式 | 注入模式 |
| **新消息处理** | 必须重启 | 忙时注入 |
| **体验问题** | 容易卡顿 | 更像"在线同事" |
