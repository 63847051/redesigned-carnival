## 🎯 项目概述

MCP-S（Multi-Agent Collaboration System）是一个完整的多智能体协作系统，支持任务依赖调度、角色池化管理、质量门禁、Prompt 模板和完整工作流编排。

### 核心特性

✅ **DAG 调度器** - 自动解析任务依赖关系
✅ **角色池管理** - 动态创建和复用 Agent 角色
✅ **质量门禁** - 自动化质量检查和控制
✅ **Prompt 模板** - 标准化 Prompt 管理
✅ **完整工作流** - 端到端的多 Agent 协作

### 3. 创建第一个工作流

```python
from mcp_workflow import create_simple_workflow, Task

# 创建任务
tasks = [
    Task("t1", "数据采集", "agent_a"),
    Task("t2", "数据清洗", "agent_b", dependencies=["t1"]),
    Task("t3", "数据分析", "agent_c", dependencies=["t2"]),
]

# 创建并执行工作流
workflow = create_simple_workflow("my_workflow", "我的工作流", tasks)
result = await workflow.execute()

print(f"完成: {result.tasks_completed}")
print(f"耗时: {result.total_execution_time:.2f}秒")
```

## 🎓 设计模式

### DP-CC-001: DAG 调度模式
- **问题**: 如何管理任务依赖关系？
- **解决**: 使用 DAG（有向无环图）和拓扑排序
- **优势**: 自动解析依赖，确保正确执行顺序

### DP-CC-002: 角色池模式
- **问题**: 如何高效管理多个 Agent 角色？
- **解决**: 使用池化管理，动态创建和复用角色
- **优势**: 减少创建开销，提高资源利用率

### DP-CC-003: 质量门禁模式
- **问题**: 如何确保输出质量？
- **解决**: 自动化质量检查和阻塞机制
- **优势**: 质量控制自动化，可配置

### DP-CC-004: 模板引擎模式
- **问题**: 如何标准化 Prompt？
- **解决**: 使用模板和变量插值
- **优势**: Prompt 复用，一致性高

### DP-CC-005: 工作流编排模式
- **问题**: 如何协调多个 Agent 协作？
- **解决**: 整合所有组件，端到端编排
- **优势**: 完整的协作解决方案
