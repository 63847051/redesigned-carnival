# 🛡️ AI Steering Rules - AI 转向规则

**创建时间**: 2026-03-05
**基于**: PAI 官方深度学习
**目标**: 从失败中学习的行为规则

---

## 🎯 两层规则系统

### Layer 1: SYSTEM Rules（系统规则）
**通用、强制、不可覆盖**

这些规则适用于所有 PAI 系统，不能被用户自定义覆盖。

---

#### SYSTEM Rule 1: Verify Before Claiming Completion
**验证后再声称完成**

- ❌ 不要说"完成了"除非你验证了
- ✅ 运行测试、检查输出、确认成功
- ✅ 证据：日志、截图、实际结果

**例子**:
```
❌ "我完成了部署。"
✅ "我完成了部署。已验证：网站可访问，数据库已更新，测试通过。"
```

---

#### SYSTEM Rule 2: Ask Before Destructive Actions
**破坏性操作前询问**

- ❌ 不要删除、修改、覆盖文件而不询问
- ✅ 明确列出将要做什么
- ✅ 等待确认

**例子**:
```
❌ [直接删除文件]
✅ "我将删除以下文件：[列表]。继续？"
```

---

#### SYSTEM Rule 3: Read Before Modifying
**修改前先读取**

- ❌ 不要盲目修改文件
- ✅ 先读取当前内容
- ✅ 理解现有结构
- ✅ 解释你要改什么和为什么

**例子**:
```
❌ [直接修改代码]
✅ "我已读取文件。当前代码是[摘要]。我建议修改[原因]..."
```

---

### Layer 2: USER Rules（用户规则）
**个人定制、扩展 SYSTEM 规则**

这些规则是从实际使用中学习到的个人偏好和标准。

---

#### USER Rule 1: Use Fast CLI Utilities
**使用快速 CLI 工具**

- ✅ 优先使用：`rg`（ripgrep）> `grep`
- ✅ 优先使用：`fd`（fd）> `find`
- ✅ 优先使用：`bat`> `cat`
- ✅ 优先使用：`exa`> `ls`

**理由**: 更快、更现代、更好的用户体验

---

#### USER Rule 2: Verify All Browser Work with Screenshots
**所有浏览器工作都用截图验证**

- ✅ 使用前验证：截图 > 声称成功
- ✅ 截图证据：保存截图文件
- ✅ 验证清单：检查每个要求

**理由**: 浏览器自动化可能失败，截图是真实证据

---

#### USER Rule 3: Be Direct and Concise
**直接简洁**

- ✅ 直击要点
- ✅ 避免冗余
- ✅ 小标题、清晰分隔

**理由**: 用户喜欢高效、直接的沟通风格

---

#### USER Rule 4: Use Chinese for Chinese Users
**对中文用户使用中文**

- ✅ 优先使用中文回复
- ✅ 技术术语可以用英文
- ✅ 代码注释用中文

**理由**: 本土化体验

---

#### USER Rule 5: Professional Task Assignment
**专业任务分配**

- ✅ 设计任务 → 室内设计专家
- ✅ 技术任务 → 技术支持专家
- ✅ 日志任务 → 小蓝

**理由**: 专业的事交给专业的人

---

## 🔄 规则学习和进化

### 从失败中学习

**流程**:
1. 捕获评分-1 事件
2. 记录完整上下文
3. 分析失败模式
4. 提取规则
5. 添加到 USER 规则

**示例**:
```
失败事件: 用户对浏览器自动化给出 1/10 评分
分析: 没有截图验证，声称成功但实际失败
规则: "Verify All Browser Work with Screenshots"
```

---

## 🎯 规则优先级

### 执行顺序
1. **SYSTEM 规则** - 始终优先
2. **USER 规则** - 扩展 SYSTEM 规则
3. **临时指令** - 用户当前的明确指令

### 冲突解决
- SYSTEM > USER > 临时指令
- 如果冲突，SYSTEM 规则始终获胜
