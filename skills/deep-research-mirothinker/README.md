# Deep Research MiroThinker Skill

## 技能简介

`deep-research-mirothinker` 是一个专业的深度研究助手技能，专为需要进行系统性、多角度、交叉验证的研究任务而设计。该技能利用 MiroThinker 的长链推理能力结合网页搜索，为用户提供扎实、全面的研究报告。

**核心升级（v2.0）**：整合了李继刚"八刀"解剖法，对任何概念进行八个维度的深度解剖，最后压缩成一句顿悟。

## 核心理念

> **用一本书的厚度讲一个概念，最后压成一句顿悟。**

## 核心特性

- 🔍 **深度分析**: 通过多步骤推理构建系统性分析
- 🌐 **多源信息**: 整合网页搜索、专业数据等多源信息
- 🔄 **交叉验证**: 多角度验证信息的可靠性和准确性
- 📊 **结构化输出**: 生成清晰、易读的研究报告
- 🎯 **智能识别**: 自动识别深度研究需求
- 🗡️ **八刀解剖**: 八个维度深度解剖核心概念（源自李继刚 ljg-learn）
- 💎 **压缩顿悟**: 从海量信息中提炼核心洞察

## 🗡️ 八刀解剖法

源自李继刚 ljg-learn 的概念解剖法，对核心概念进行八个维度的深度解剖：

| 刀 | 维度 | 核心问题 |
|----|------|----------|
| 第一刀 | 历史 | 最早从哪冒出来 → 怎么变的 → 哪一步拐成了今天的意思 |
| 第二刀 | 辩证 | 它的反面是什么 → 正反碰撞后，更高一层的理解是什么 |
| 第三刀 | 现象 | 扔掉所有预设，回到事情本身 → 用一个日常场景把它还原出来 |
| 第四刀 | 语言 | 拆字源 → 画出相邻概念的语义网 → 这个词暗含什么隐喻 |
| 第五刀 | 形式 | 写一个公式或形式化表达 → 公式在哪里失效 |
| 第六刀 | 存在 | 这个概念改变了人怎么活着 |
| 第七刀 | 美感 | 它美在哪？用一个具体意象呈现 |
| 第八刀 | 元反思 | 我们在用什么隐喻理解它？这个隐喻挡住了什么？换一个会怎样 |

## 📝 研究流程

### 定锚
1. 核心概念的最通行定义是什么？常见误解在哪？
2. 概念里藏着哪几个核心词素？

### 八刀
八个方向各切一刀。每刀 2-3 句，只留筋骨。

### 内观
1. 变成这个概念本身，用第一人称看世界。3-5 句。
2. 八刀之中，哪几刀指向同一个深层结构？把它提出来。

### 压缩
1. **公式**：`概念 = ...`
2. **一句话**：用最简单的话说出最深的理解
3. **结构图**：纯 ASCII 画出概念的骨架

## 🎯 适用场景

### 最佳适用场景
- 技术趋势研究
- 市场分析报告
- 竞争对手调研
- 技术方案对比
- 学术文献综述
- 行业发展分析
- 概念深度理解
- 原理性研究

### 不适用场景
- 简单事实查询
- 个人主观观点
- 实时性要求极高的信息
- 需要专业领域深度知识的问题

## ⚡ 性能特点

### 优势
- 全面性：覆盖多个信息源和角度
- 深度性：通过长链推理进行深度分析
- 系统性：结构化的分析框架
- 顿悟性：从海量信息中提炼核心洞察
- 实时性：基于最新网络信息

### 限制
- 复杂研究需要较长时间
- 依赖网络信息质量
- 某些专业领域可能需要人工验证
- 大量数据处理可能有延迟

## 🔧 使用建议

### 对用户的建议
1. **明确研究目标**: 在提出问题时明确研究目的和范围
2. **提供背景信息**: 如有相关背景信息，有助于更精准的研究
3. **设定合理期望**: 深度研究需要时间，请耐心等待结果
4. **验证关键信息**: 对重要结论建议进行人工验证

### 对开发者的建议
1. **持续优化**: 根据使用反馈持续优化研究算法
2. **信息源扩展**: 逐步增加专业信息源和数据库
3. **性能优化**: 优化长时间研究的用户体验
4. **质量控制**: 建立信息质量评估机制
