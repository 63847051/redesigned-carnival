## 🎯 技能概述

### **核心目标**
> "Ship interfaces that feel deliberate, premium, and current."

### **适用场景**
- 视觉强度高的落地页
- 网站、应用原型
- 演示、游戏 UI
- 需要艺术指导的项目

### **关键原则**
- ✅ 构图优先 > 组件数量
- ✅ 约束 > 自由
- ✅ 意图 > 装饰
- ✅ 少即是多

---

## 📋 工作模型（三件事）

### **开始前先写**：

#### **1. Visual Thesis（视觉论题）**
```
一句话描述：情绪、材质、能量
示例："温暖、木质、平静的能量"
```

#### **2. Content Plan（内容计划）**
```
Hero → Support → Detail → Final CTA
```

#### **3. Interaction Thesis（交互论题）**
```
2-3 个动效想法
示例：Hero 入场、滚动链接、悬停揭示
```

## 🎨 核心规则

### **Landing Pages**

#### **Hero 规则** ⭐⭐⭐
- ✅ 全屏 edge-to-edge
- ✅ 品牌第一，标题第二
- ❌ 无 Hero 卡片
- ❌ 无统计条、Logo 云

#### **全屏规则** ⭐⭐⭐
```css
/* 错误方式 */
.hero {
  max-width: 1200px;
  margin: 0 auto;
}

/* 正确方式 */
.hero {
  width: 100%;
  padding: 0;
}

.hero-content {
  max-width: 600px;  /* 只限制内容 */
  margin: 0 auto;
}
```

### **Apps（Linear 风格）**
- ✅ 平静的表面层级
- ✅ 强排版和间距
- ✅ 少颜色
- ✅ 密集但可读
- ❌ 无仪表板卡片马赛克

### **Imagery**
- ✅ 至少一个强真实图像
- ✅ 现场摄影 > 抽象渐变
- ❌ 无内置 UI 框架

### **Copy**
- ✅ 产品语言 > 设计评论
- ✅ 标题承担意义
- ❌ 无填充文案

### **Utility Copy（产品 UI）**
- ✅ "Selected KPIs"
- ❌ "Unlock your potential"

---

## 🔒 硬规则（Hard Rules）

### **绝对禁止**：
- ❌ 默认无卡片
- ❌ 默认无 Hero 卡片
- ❌ 全屏 Hero 无盒子/中心列
- ❌ 每区域不超过一个主导想法
- ❌ 品牌页面标题不压倒品牌
- ❌ 无填充文案
- ❌ 无分割屏 Hero（除非文本在平静侧）
- ❌ 无理由超过两种字体
- ❌ 无理由超过一个强调色

---

## ✅ 石蕊测试（Litmus Checks）

### **设计前问自己**：
1. ✅ 品牌在第一屏 unmistakable？
2. ✅ 有一个强视觉锚点？
3. ✅ 只扫描标题能理解页面？
4. ✅ 每个区域有一个工作？
5. ✅ 卡片真的必要？
6. ✅ 动效改进层级或氛围？
7. ✅ 移除装饰阴影仍感觉 premium？

## 💡 最佳实践

### **1. 从构图开始**
```
❌ 错误：先选择组件库
✅ 正确：先画构图草图
```

### **2. 限制系统**
```
❌ 错误：3-4 种字体，多个强调色
✅ 正确：2 种字体，1 个强调色
```

### **3. 删除不必要**
```
❌ 错误：添加装饰元素
✅ 正确：删除直到无法删除
```
