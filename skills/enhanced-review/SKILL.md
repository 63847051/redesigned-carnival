## 📖 概述

增强审查技能实施**两阶段审查机制**：先审查规范合规性，再审查代码质量。关键问题会阻止进度。

### 核心原则

1. **两阶段审查** - 规范合规 + 代码质量
2. **严重度分级** - Critical/High/Medium/Low
3. **关键问题阻止** - Critical 问题必须修复
4. **建设性反馈** - 提供具体的改进建议
5. **审查清单** - 标准化的审查项目

## 🔄 两阶段审查流程

### Phase 1: 规范合规性审查

**目标**：检查是否符合任务规范

```markdown
## Phase 1: 规范合规性审查

### 任务规范回顾
- **任务**: XXX
- **要求**: XXX
- **文件**: XXX
- **验证标准**: XXX

### 检查项
- [ ] 是否完成了所有要求的功能？
- [ ] 是否使用了指定的文件路径？
- [ ] 是否符合指定的技术栈？
- [ ] 是否通过了指定的验证步骤？

### 结果
✅ 通过 / ❌ 不通过
```

### Phase 2: 代码质量审查

**目标**：检查代码质量

```markdown
## Phase 2: 代码质量审查

### 代码质量检查
- [ ] 代码可读性（命名、注释、结构）
- [ ] 代码复用性（DRY 原则）
- [ ] 错误处理（异常、边界情况）
- [ ] 性能问题（时间复杂度、内存）
- [ ] 安全问题（输入验证、敏感数据）
- [ ] 测试覆盖（单元测试、集成测试）

### 结果
✅ 通过 / ❌ 不通过
```

## 🚨 问题严重度分级

### Critical（关键）- 阻止进度

```markdown
**问题**: XXX

**严重度**: 🔴 Critical

**影响**:
- 功能无法正常工作
- 安全漏洞
- 数据丢失风险

**必须修复**:
- 立即修复
- 修复后重新审查

**阻塞**: 是，必须修复才能继续
```

**示例**：
- SQL 注入漏洞
- 空指针异常
- 数据库连接未关闭
- 敏感数据明文存储

### High（高）- 强烈建议修复

```markdown
**问题**: XXX

**严重度**: 🟠 High

**影响**:
- 功能部分受限
- 性能严重下降
- 潜在的安全风险

**建议**:
- 尽快修复
- 修复后验证

**阻塞**: 否，但强烈建议修复
```

**示例**：
- 缺少错误处理
- 性能瓶颈（O(n²) 可以优化为 O(n)）
- 硬编码配置
- 缺少日志记录

### Medium（中）- 建议修复

```markdown
**问题**: XXX

**严重度**: 🟡 Medium

**影响**:
- 代码可维护性
- 代码可读性

**建议**:
- 时间允许时修复

**阻塞**: 否
```

**示例**：
- 变量命名不清晰
- 缺少注释
- 代码重复
- 过长的函数（> 50 行）

### Low（低）- 可选修复

```markdown
**问题**: XXX

**严重度**: 🟢 Low

**影响**:
- 代码风格
- 微小的优化

**建议**:
- 有时间时优化

**阻塞**: 否
```

**示例**：
- 空格不一致
- 未使用的变量
- 可以简化的逻辑

## Phase 2: 代码质量审查

### 结果: ✅ 通过 / ❌ 不通过

### 发现的问题

#### 🔴 Critical（1 个）

**1. SQL 注入漏洞**

**文件**: `src/api/user.ts`
**行号**: 45
**代码**:
```typescript
const query = `SELECT * FROM users WHERE id = ${userId}`;
```

**问题**: 直接拼接 SQL，存在注入风险
**修复**:
```typescript
const query = `SELECT * FROM users WHERE id = ?`;
await db.query(query, [userId]);
```

**必须修复**: ✅ 是

---

#### 🟠 High（2 个）

**1. 缺少错误处理**

**文件**: `src/api/stock.ts`
**行号**: 23
**代码**:
```typescript
const data = await fetchStockData(symbol);
return data.price;
```

**问题**: 如果 fetch 失败，会抛出未捕获的异常
**修复**:
```typescript
try {
  const data = await fetchStockData(symbol);
  return data.price;
} catch (error) {
  logger.error('Failed to fetch stock data', error);
  throw new Error('Unable to fetch stock data');
}
```

**建议**: 尽快修复

---

**2. 性能问题**

**文件**: `src/processor/data.ts`
**行号**: 67
**代码**:
```typescript
for (let i = 0; i < data.length; i++) {
  for (let j = 0; j < data.length; j++) {
    // O(n²) 复杂度
  }
}
```

**问题**: 嵌套循环导致 O(n²) 复杂度
**修复**: 使用 Map 或对象优化

**建议**: 尽快修复

## 💡 最佳实践

### ✅ 推荐做法

1. **客观公正**
   - 基于事实，不基于个人偏好
   - 使用审查清单

2. **建设性反馈**
   - 提供具体的改进建议
   - 解释为什么这样改

3. **优先级明确**
   - Critical 问题优先
   - 提供修复建议

4. **及时审查**
   - 任务完成后立即审查
   - 不要拖延

### ❌ 避免做法

1. **过于严格**
   - 不要吹毛求疵
   - 允许不同的代码风格

2. **过于宽松**
   - 不能放任 Critical 问题
   - 要坚持质量标准

3. **主观批评**
   - 不要说"这代码很烂"
   - 要说"这里可以改进"
