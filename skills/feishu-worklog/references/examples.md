# 小蓝使用示例

## 常见对话示例

### 快速记录

```
用户: 小蓝,记录一下今天整理茶水柜
小蓝: ✅ 已记录: 整理茶水柜
       类型: 现场 | 优先级: 普通重要 | 状态: 待完成

---

用户: 小蓝,完成了会议室的设计方案
小蓝: ✅ 已记录: 会议室设计方案
       类型: 设计 | 优先级: 普通重要 | 状态: 已完成

---

用户: 小蓝,紧急处理办公室电路故障
小蓝: ✅ 已记录: 处理办公室电路故障
       类型: 机电 | 优先级: 第一优先 | 状态: 待完成
```

### 查询记录

```
用户: 小蓝,今天的任务有哪些?
小蓝: 📋 今天有 3 条记录:
       1. 整理茶水柜 - 现场 - 待完成
       2. 会议室设计方案 - 设计 - 已完成
       3. 电路维修 - 机电 - 待完成

---

用户: 小蓝,搜索所有设计相关的任务
小蓝: 📋 找到 5 条设计相关记录:
       1. 会议室设计方案 - 设计 - 已完成
       2. 接待区效果图 - 设计 - 待完成
       ...

---

用户: 小蓝,有哪些待完成的任务?
小蓝: 📋 待完成任务共 2 条:
       1. 整理茶水柜 - 现场 - 普通重要
       2. 电路维修 - 机电 - 第一优先
```

### 更新状态

```
用户: 小蓝,把茶水柜任务标记为完成
小蓝: ✏️ 已更新: 整理茶水柜
       状态: 待完成 → 已完成
       完成时间: 2026/03/03

---

用户: 小蓝,修改电路维修为重要优先级
小蓝: ✏️ 已更新: 电路维修
       优先级: 普通重要 → 重要

---

用户: 小蓝,给会议室设计加个备注:客户确认通过
小蓝: ✏️ 已更新: 会议室设计方案
       备注: 客户确认通过
```

### 删除记录

```
用户: 小蓝,删除茶水柜那条记录
小蓝: ⚠️ 找到记录: 整理茶水柜 (现场-待完成)
       确认删除吗? (回复"确认"执行)

用户: 确认
小蓝: 🗑️ 已删除: 整理茶水柜
```

## 批量操作

```
用户: 小蓝,把所有待完成的设计任务标记为重要
小蓝: 找到 3 条待完成的设计任务,确认全部更新为重要优先级吗?

用户: 确认
小蓝: ✅ 已批量更新 3 条记录
```

## 自然语言理解示例

### 模糊表达
```
"今天做了什么" → 查询今天的记录
"那个茶水柜的任务怎么样了" → 搜索茶水柜相关记录
"还有哪些没做完的" → 查询待完成任务
"之前说的电路维修" → 搜索电路维修记录
```

### 同义词识别
```
记录/添加/新建/记一下 → 创建记录
查询/搜索/看看/找 → 查询记录
更新/修改/改成/标记为 → 更新记录
删除/移除/去掉 → 删除记录
完成/搞定/弄好 → 状态改为已完成
```

### 智能推断
```
"今天画了会议室效果图" → 类型=设计
"刚修好办公室的灯" → 类型=机电
"整理了茶水柜" → 类型=现场
"正在装修墙面" → 类型=施工
```

## 错误处理

```
用户: 小蓝,删除那个任务
小蓝: 找到多条记录,请指定:
       1. 整理茶水柜
       2. 清洁会议室
       请选择序号或提供更多关键词

---

用户: 小蓝,完成任务
小蓝: 未找到匹配的记录,请提供具体任务名称

---

用户: 小蓝,查询明天的任务
小蓝: 明天还没有任务记录,需要添加吗?
```
