## 📊 分层架构设计

### L0: 原始数据层（Raw Data）
- **位置**: `memory/YYYY-MM-DD.md`
- **内容**: 完整的每日对话记录
- **用途**: 深度检索时的原始素材
- **保留策略**: 永久保留

### L1: 关键点层（Key Points）
- **位置**: `memory/key-points/YYYY-MM.md`
- **内容**: 每月的关键事件、决策、行动项
- **用途**: 快速回顾某个时期的关键信息
- **保留策略**: 按月保留，12 个月后压缩

### L2: 结构化知识层（Structured Knowledge）
- **位置**: `memory/structured/{people,projects,knowledge,preferences}/`
- **内容**: 按主题分类的结构化知识
- **用途**: 针对性检索特定类型的知识
- **保留策略**: 重要知识永久保留

### L3: 长期洞察层（Long-term Insights）
- **位置**: `MEMORY.md`
- **内容**: 跨时间的长期洞察和模式
- **用途**: 系统级记忆，每次会话自动加载
- **保留策略**: 永久保留，持续更新

## 💡 核心洞察

### 洞察 1: 记忆是分层的
> **"不同层级的记忆有不同的用途和价值。"**

- **L0**（原始数据）: 深度检索时才用
- **L1**（关键点）: 快速回顾
- **L2**（结构化）: 针对性检索
- **L3**（洞察）: 每次会话必用

### 洞察 2: Token 优化是分层的
> **"优先加载高价值记忆，按需加载低价值记忆。"**

加载策略：
1. **L3** 永远保留（每次会话自动加载）
2. **L2** 优先保留（相关主题时加载）
3. **L1** 按需保留（回顾特定时期时加载）
4. **L0** 深度检索时才用（不预加载）

### 洞察 3: 记忆需要主动管理
> **"记忆不是被动的存储，而是主动的计算和提取。"**

自动化流程：
1. **对话结束时** → 提取关键点（L1）
2. **每日结束时** → 分类结构化知识（L2）
3. **每周结束时** → 提炼长期洞察（L3）
4. **每月结束时** → 压缩和归档
