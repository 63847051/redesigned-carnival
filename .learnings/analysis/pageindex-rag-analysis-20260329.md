## 🎯 核心理念

### 🔴 **关键洞察**

**PageIndex 是一种"无向量 RAG"**：
- ❌ **不使用向量数据库**
- ✅ **使用第一性原理推理**
- ✅ **自底向上的摘要生成**

### 📊 **核心流程**

```
1. 解析文档（parse_document）
   → 树形结构（段落、章节）
   
2. 构建摘要（build_summaries）
   → LLM 生成每个节点的摘要
   
3. 保存索引（storage.save）
   → JSON 格式，包含树形结构
   
4. 检索（retrieve）
   → 基于推理，不依赖向量相似度
   
5. 生成答案（LLM）
   → 使用检索到的上下文
```
