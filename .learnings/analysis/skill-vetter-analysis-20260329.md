## 🎯 核心功能

### 📋 **Skill Vetter** - 技能安全审查协议

**核心理念**：

**使用场景**：
- ✅ 从 ClawdHub 安装技能前
- ✅ 从 GitHub 仓库运行技能前
- ✅ 评估其他 Agent 分享的技能
- ✅ 安装未知代码前

---

## 🔍 **审查流程（4 步）**

### Step 1: 来源检查
**问题清单**：
- [ ] 这个技能来自哪里？
- [ ] 作者是否已知/有声誉？
- [ ] 下载量/星标数是多少？
- [ ] 最后更新时间？
- [ ] 有其他 Agent 的评价吗？

### Step 2: 代码审查（必须）⭐
**阅读所有文件，检查红旗**：

🚨 **立即拒绝，如果看到**：
- curl/wget 到未知 URL
- 向外部服务器发送数据
- 请求凭证/Token/API 密钥
- 读取 ~/.ssh, ~/.aws, ~/.config（无明显原因）
- 访问 MEMORY.md, USER.md, SOUL.md, IDENTITY.md
- 使用 base64 解码任何内容
- 使用 eval() 或 exec() 处理外部输入
- 修改工作区外的系统文件
- 安装包但不列出
- 向 IP 而非域名发起网络调用
- 混淆代码（压缩、编码、最小化）
- 请求提升/sudo 权限
- 访问浏览器 cookies/sessions
- 触摸凭证文件

### Step 3: 权限范围
**评估**：
- [ ] 需要读取什么文件？
- [ ] 需要写入什么文件？
- [ ] 运行什么命令？
- [ ] 需要网络访问吗？到哪里？
- [ ] 范围是否最小化？

### Step 4: 风险分类
| 风险级别 | 示例 | 操作 |
|---------|------|------|
| 🟢 LOW | 笔记、天气、格式化 | 基本审查，可以安装 |
| 🟡 MEDIUM | 文件操作、浏览器、API | 完整代码审查 |
| 🔴 HIGH | 凭证、交易、系统 | 需要人工批准 |
| ⛔ EXTREME | 安全配置、root 访问 | **不要安装** |
