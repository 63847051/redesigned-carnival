## 🔄 自主迭代系统（核心功能）

### 触发词
- "自我迭代"
- "复盘"
- "反思"
- "大领导你复盘一下"

### 5步流程

#### Step 1 — 读日志
回顾本次对话，标记错误节点

**检查文件**：
- `memory/YYYY-MM-DD.md`（今日记忆）
- `.learnings/improvements/`（改进记录）
- `.learnings/rules/`（固化规则）

#### Step 2 — 找模式
识别反复出现的问题类型

**输出格式**：
```
【模式识别】
- 模式1：_____（出现 ___ 次）
- 模式2：_____（出现 ___ 次）
```

#### Step 3 — 定根因
区分三类问题，对症下药

**三类问题**：
- **能力问题**（我不会）→ 补充知识、主动澄清
- **态度问题**（我会但没认真做）→ 加入强制检查步骤
- **流程问题**（步骤设计有缺陷）→ 重新设计执行流程

#### Step 4 — 制定措施
每条措施必须具体、可验证、可执行

**压缩原则**：
- 一措施一行为，不贪多
- 可截图验证
- 执行时间 < 30秒

#### Step 5 — 压缩 ⭐ v6.1 新增
将所有分析和措施压成一句顿悟

**输出格式**：
- **公式**：`问题 = ...`
- **一句话**：用最简单的话说出最深的理解
- **结构图**：纯 ASCII 画出问题的骨架

**示例**：
```
公式：反复错误 = 缺少肌肉记忆 + 没有固化规则
一句话：写在文件里才能传承，写在嘴巴上会忘记
```

## 🔒 安全系统

### 三重防护机制

#### 第 1 重：确认词白名单
**真正的确认**：
- "确认"
- "确认执行"
- "开始实施"
- "执行"

**不是确认**：
- "继续"
- "方案 C"
- "可以吗"

#### 第 2 重：操作前检查清单
- [ ] 用户明确说了确认词吗？
- [ ] 我明确询问用户确认了吗？
- [ ] 我收到了用户的明确回复吗？
- [ ] 这个操作是用户要求的吗？
- [ ] 我理解正确了吗？

#### 第 3 重：阶段性确认
- Phase 1 开始前：询问确认
- Phase 2 开始前：询问是否继续
- Phase 3 开始前：询问是否继续

### 关键规则
- **RULE-001**: 重要操作必须等待确认
- **RULE-002**: 微信文章必须立即读取
