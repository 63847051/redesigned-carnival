## 🚨 规则内容

### 核心规则
> **"重要操作必须等待用户明确确认，绝不擅自执行。"**

### 确认词白名单（只有这些才是真正的确认）

**有效确认词**：
- ✅ "确认"
- ✅ "确认执行"
- ✅ "开始实施"
- ✅ "执行"

**❌ 以下绝对不是确认**：
- ❌ "继续"
- ❌ "方案 C"
- ❌ "可以吗"
- ❌ "好的"
- ❌ "明白了"

### 1️⃣ Git 操作
- git push

### 4️⃣ API 调用
- 发送重要消息
- 创建/删除资源
- 修改重要数据

## ✅ 操作前强制检查清单

**三重防护机制**：

### 第 1 重：确认词白名单
- [ ] 用户明确说了确认词吗？
- [ ] 确认词在白名单中吗？

### 第 2 重：操作前检查清单
- [ ] 我明确询问用户确认了吗？
- [ ] 我收到了用户的明确回复吗？
- [ ] 回复中包含有效的确认词吗？

### 第 3 重：阶段性确认机制
- [ ] 每个阶段都要重新确认
- [ ] 不假设"用户之前同意，现在也同意"

## 🎯 正确的确认流程

### ❌ 错误示例
```
我: 我要删除这些文件，可以吗？
用户: 继续
我: 好的，开始删除...  ❌ 错误！"继续"不是确认词
```

### ✅ 正确示例
```
我: 我要删除这些文件，请确认是否执行？
用户: 确认执行
我: 收到，开始执行...  ✅ 正确！"确认执行"是确认词
```

## 🔒 检查脚本

**文件**: `scripts/confirmation-check.sh`

```bash
#!/bin/bash
# 确认词检查脚本

check_confirmation() {
    local user_input="$1"

    # 有效确认词列表
    local confirm_words=("确认" "确认执行" "开始实施" "执行")

    # 检查是否包含确认词
    for word in "${confirm_words[@]}"; do
        if [[ "$user_input" == *"$word"* ]]; then
            return 0  # 找到确认词
        fi
    done

    return 1  # 没有找到确认词
}

# 使用示例
if check_confirmation "$USER_INPUT"; then
    echo "✅ 用户已确认"
else
    echo "❌ 用户未确认，请等待明确确认"
fi
```

## 💡 根本原因

### 为什么需要这个规则？

1. **防止误操作**
   - 重要操作不可逆
   - 用户需要时间考虑

2. **尊重用户意愿**
   - 用户可能改变主意
   - 需要明确的同意信号

3. **避免误解**
   - "继续"可能只是表示"我在听"
   - "可以吗"是疑问，不是同意
