## 🎯 进化目标

解决"规则记录了但执行不好"的核心问题：

1. 记录是静态的 → 存在文件里
2. 执行是动态的 → 依赖每次对话时的判断
3. 缺乏强制检查 → 没有硬编码的检查点
4. 会话记忆问题 → 每次"醒来"可能忘记检查
5. 理解偏差 → 把询问当成指令

---

## 🚀 进化内容

### 1. 系统提示词配置

**文件**: `.learnings/rules/RULE-SYSTEM-PROMPT.md`
**配置**: `openclaw.json` 的 `systemPrompt` 字段
**Token**: 109 tokens
**功能**: 每次对话自动加载关键规则

**关键内容**:
- RULE-001: 重要操作必须等待确认
- 确认词白名单（确认、确认执行、开始实施、执行）
- 歧义检测规则（疑问句特征）
- 5项检查清单

**解决问题**: 1（记录静态）、4（会话记忆）

---

### 2. 歧义检测脚本

**文件**: `scripts/ambiguity-detector.sh`
**Token**: 50 tokens
**功能**: 自动检测疑问句和不完整指令

**测试结果**:
```bash
# 测试疑问句
输入: "方案 C，可以并行的处理任务吗"
输出: ⚠️ 检测到疑问句 ✅

# 测试确认词
输入: "确认执行"
输出: ✅ 无歧义 ✅
```

**解决问题**: 2（动态判断）、5（理解偏差）

---

### 3. 操作前检查脚本

**文件**: `scripts/pre-operation-check-simple.sh`
**Token**: 100 tokens
**功能**: 强制5项检查清单

**检查清单**:
1. 用户明确说了确认词吗？
2. 我明确询问用户确认了吗？
3. 我收到了用户的明确回复吗？
4. 这个操作是用户要求的吗？
5. 我理解正确了吗？

**解决问题**: 3（缺乏强制检查）
