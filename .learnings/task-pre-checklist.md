**版本**: v1.0
**创建时间**: 2026-03-24 08:40

---

## ✅ 每次任务前必做

在开始任何任务前，必须完成以下检查：

---

### 📚 规则检查

- [ ] **已读取固化规则库**
  - 检查 `.learnings/rules/consolidated-rules-database.md`
  - 确认是否有与本次任务相关的规则
  - 如果有，严格遵守

---

### 🔍 场景检查

- [ ] **检查是否属于历史高错误率场景**
  - opencode 模型任务 → 使用 OpenCode CLI
  - 微信文章读取 → 使用 read-wechat.py
  - 重要操作（Git推送、文件删除）→ 等待用户确认

---

### 💬 需求确认

- [ ] **需求是否清晰？**
  - 不清晰 → 先澄清再执行
  - 有歧义 → 主动询问
  - 不确定 → 不要假设

---

### 🎯 质量检查

- [ ] **输出前自查质量**
  - 内容是否完整？
  - 格式是否正确？
  - 是否符合用户要求？
  - 是否有明显错误？

---

## 🚨 常见错误场景

### 场景 1: 分配给小新任务
**检查清单**：
- [ ] 这是技术任务吗？
- [ ] 我要用 OpenCode CLI 吗？
- [ ] 命令是 `opencode -m opencode/minimax-m2.5-free run` 吗？

**错误做法**：
```bash
sessions_spawn -runtime subagent -model opencode/minimax-m2.5-free
```

**正确做法**：
```bash
opencode -m opencode/minimax-m2.5-free run "任务"
```

---

### 场景 2: 读取微信文章
**检查清单**：
- [ ] 这是 mp.weixin.qq.com 链接吗？
- [ ] 我有3种方法可以读取吗？
- [ ] 我立即行动了吗？

**错误做法**：
```
"读不了"
"无法读取"
```

**正确做法**：
```bash
python3 /root/.openclaw/workspace/scripts/read-wechat.py "URL"
```

---

### 场景 3: 重要操作
**检查清单**：
- [ ] 这是重要操作吗？（Git推送、文件删除等）
- [ ] 用户说了确认词吗？
- [ ] 我收到明确回复了吗？

**确认词白名单**：
- ✅ "确认"
- ✅ "确认执行"
- ✅ "开始实施"
- ✅ "执行"

**非确认词**：
- ❌ "继续"
- ❌ "方案 C"
- ❌ "可以吗"
