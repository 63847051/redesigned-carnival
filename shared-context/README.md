# 共享上下文目录

**版本**: v1.0  
**创建时间**: 2026-03-18  
**目的**: 跨 Agent 的任务和项目信息共享

---

## 📁 目录结构

```
shared-context/
├── README.md              # 本文件
├── tasks/                 # 共享任务列表
│   ├── README.md
│   ├── blue-light地标.yaml    # 蓝色光标项目任务
│   └── template.yaml           # 任务模板
└── projects/              # 项目信息
    ├── README.md
    ├── blue-light地标.md       # 蓝色光标项目详情
    └── template.md             # 项目模板
```

---

## 🎯 设计原则

### 1. 单一数据源
- 每个任务只有一个 YAML 文件
- 避免信息分散和重复
- 所有 Agent 读取同一个文件

### 2. 标准化格式
- 使用 YAML 格式（易读易解析）
- 统一字段名称
- 版本控制友好

### 3. 最小化 Token
- 只包含必要字段
- 避免冗余描述
- 支持 DP-006 子 Agent 优化

## 🔄 使用流程

### 创建任务
```bash
# 复制模板
cp shared-context/tasks/template.yaml shared-context/tasks/TASK-001.yaml

# 编辑任务信息
vim shared-context/tasks/TASK-001.yaml
```

### 更新任务
```bash
# 编辑任务文件
vim shared-context/tasks/TASK-001.yaml

# 更新 updated 字段
# 修改 status 和 progress
```

### 读取任务
```bash
# Agent 读取任务
cat shared-context/tasks/TASK-001.yaml

# 或使用 Python
python3 -c "import yaml; print(yaml.safe_load(open('shared-context/tasks/TASK-001.yaml')))"
```

## 🤝 跨 Agent 协作

### 室内设计专家
- **读取**: 设计任务详情
- **更新**: 任务状态、进度
- **写入**: 设计结果

### 💻 小新（技术支持专家）
- **读取**: 技术任务详情
- **更新**: 任务状态、进度
- **写入**: 代码和脚本

### 📋 小蓝（工作日志管理专家）
- **读取**: 日志任务详情
- **更新**: 任务状态
- **写入**: 日志记录

### 🎯 大领导（主控 Agent）
- **读取**: 所有任务
- **创建**: 新任务
- **分配**: 任务到专家
- **监督**: 进度和质量
- **汇总**: 成果汇报

## 💡 最佳实践

### 1. 任务命名规范
- 使用有意义的前缀: `BL-001` (蓝色光标), `TECH-001` (技术)
- 任务ID唯一性
- 标题简洁明了

### 2. 状态更新及时
- 每次修改任务后更新 `updated` 字段
- 开始任务时设置 `started`
- 完成任务时设置 `completed`

### 3. 依赖关系明确
- 使用 `dependencies` 数组列出依赖
- 确保依赖任务先完成
- 避免循环依赖

### 4. 文档化决策
- 在 `notes` 中记录重要决策
- 说明任务变更原因
- 保留历史信息
