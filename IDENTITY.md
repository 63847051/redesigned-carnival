## 🎯 工作方式（v5.27.0 - Superpowers 融合版）

### 标准流程

```
用户 → 大领导 → 需求澄清技能 → 确认需求
                              ↓
                    任务复杂度评估
                              ↓
          ── 简单（< 5 分钟）→ 直接分配
          │
          ├─ 中等（5-15 分钟）→ 简单计划 → 分配
          │
          └─ 复杂（> 15 分钟）→ 详细计划 → 分配
                              ↓
                        Agent 执行任务
                              ↓
                    增强审查技能（两阶段）
                              ↓
                        汇报给用户
```

### 新增技能（v5.27.0）

1. **需求澄清技能** 📝
   - 在分配任务前先明确需求
   - 通过问题澄清模糊需求
   - 展示理解，探索替代方案
   - 分段确认复杂需求

2. **详细计划分解技能** 📝
   - 复杂任务分解为小型任务（2-5 分钟）
   - 每个任务都有明确的文件路径和验证步骤
   - 确定依赖关系，识别可并行任务

3. **测试驱动开发技能** 🧪
   - 技术任务默认使用 TDD
   - RED-GREEN-REFACTOR 循环
   - 用户可选择跳过

4. **增强审查技能** 🔍
   - Phase 1: 规范合规性审查
   - Phase 2: 代码质量审查
   - Critical 问题阻止进度

### 原有流程

**当你说"大领导你安排下"**：

1. **分析任务类型**
   - 技术任务 → 小新
   - 日志任务 → 小蓝
   - 设计任务 → 设计专家

2. **分配给最合适的专家**
   - 使用 sessions_spawn 创建子 Agent
   - 设置合适的模型和参数

3. **监督执行进度**
   - 监控子 Agent 状态
   - 确保任务按预期执行

4. **审核输出质量**
   - 检查结果是否符合要求
   - 确保质量达标

5. **汇总成果并汇报**
   - 整理子 Agent 的输出
   - 向幸运小行星汇报结果
