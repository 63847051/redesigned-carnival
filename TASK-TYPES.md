# TASK-TYPES.md - 任务类型分类手册

**版本**: v1.0  
**创建时间**: 2026-03-18  
**目的**: 提供清晰的任务分类标准，确保任务分配到最合适的专家

---

## 🎯 任务分类原则

**核心理念**: "专业的事交给专业的人"

**分类维度**:
1. **任务领域** - 设计/技术/日志/协调
2. **复杂度** - 简单/中等/复杂
3. **紧急度** - 低/中/高
4. **模型需求** - 免费/主模型/旗舰

---

## 🏠 设计任务

### 触发关键词
- **设计类**: 设计、图纸、平面图、立面图、天花、地面、排砖
- **空间类**: 会议室、茶水区、更衣室、柜体、家具
- **施工类**: 施工图、节点图、大样图

### 典型任务示例
1. "修改3F男女更衣室排砖平面图"
2. "绘制男女更衣室立面图和排版"
3. "整理所有楼层打印机柜图纸"
4. "4F会议室合并（注意天花地面）"
5. "6F茶水区天花修改"

### 负责专家
- **室内设计专家** (专业 Agent)
- **模型**: GLM-4.7 (主模型，中文优化)
- **特点**: 理解设计规范、施工标准

### 任务边界
✅ **包含**:
- CAD 图纸绘制和修改
- 平面图、立面图、剖面图
- 空间规划和布局
- 材料和工艺说明

❌ **不包含**:
- 代码编写 → 转给 💻 小新
- 日志记录 → 转给 📋 小蓝

## 📊 复杂度分级

### Level 1: 简单任务
- **特点**: 单一任务，步骤明确
- **模型**: 免费模型优先
- **示例**: "更新一条工作日志"

### Level 2: 中等任务
- **特点**: 多步骤，需要一定思考
- **模型**: 免费模型或主模型
- **示例**: "绘制一个房间的立面图"

### Level 3: 复杂任务
- **特点**: 多维度，需要深度分析
- **模型**: 主模型 GLM-4.7
- **示例**: "分析项目整体风险并制定应对方案"

---

## 🚨 紧急度分级

### 低优先级
- **响应时间**: 4 小时内
- **示例**: 长期规划、知识整理

### 中优先级
- **响应时间**: 2 小时内
- **示例**: 日常任务、常规工作

### 高优先级
- **响应时间**: 30 分钟内
- **示例**: 紧急问题、关键决策

## 🔄 任务分配流程

### Step 1: 任务识别
```
用户输入 → 关键词检测 → 任务类型判断
```

### Step 2: 专家匹配
```
任务类型 → 对应专家 → 模型选择
```

### Step 3: 执行监督
```
专家执行 → 进度跟踪 → 质量审核
```

### Step 4: 结果汇总
```
专家输出 → 大领导审核 → 向用户汇报
```

## 🎯 边界处理原则

### 跨界任务
当任务涉及多个领域时：
1. **主领域优先** - 按主要工作内容分类
2. **协调处理** - 通过大领导协调多个专家
3. **分阶段执行** - 设计 → 开发 → 记录

### 不明确任务
当任务类型不明确时：
1. **询问用户** - 确认具体需求
2. **分配给主控** - 由大领导分析
3. **临时归类** - 先按最接近的类型处理
