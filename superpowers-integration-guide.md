#### 1. 更新 IDENTITY.md

在 IDENTITY.md 中添加新的工作流程：

```markdown
## 🔄 工作流程（v5.27.0 - Superpowers 融合版）

### 标准流程

```
用户 → 大领导 → 需求澄清技能 → 确认需求
                              ↓
                    任务复杂度评估
                              ↓
          ┌─ 简单（< 5 分钟）→ 直接分配
          │
          ├─ 中等（5-15 分钟）→ 简单计划 → 分配
          │
          └─ 复杂（> 15 分钟）→ 详细计划 → 分配
                              ↓
                        Agent 执行任务
                              ↓
                    增强审查技能（两阶段）
                              ↓
                        汇报给用户
```

### 新增技能

1. **需求澄清技能**
   - 在分配任务前先明确需求
   - 通过问题澄清模糊需求
   - 展示理解，探索替代方案

2. **详细计划分解技能**
   - 复杂任务分解为小型任务（2-5 分钟）
   - 每个任务都有明确的文件路径和验证步骤
   - 确定依赖关系，识别可并行任务

3. **测试驱动开发技能**
   - 技术任务默认使用 TDD
   - RED-GREEN-REFACTOR 循环
   - 用户可选择跳过

4. **增强审查技能**
   - Phase 1: 规范合规性审查
   - Phase 2: 代码质量审查
   - Critical 问题阻止进度
```

```markdown
## 🛠️ 技能使用规则（v5.27.0 - Superpowers 融合版）

### 自动触发

**需求澄清技能**:
- 触发条件：用户提出新任务
- 触发时机：分配任务前

**详细计划分解技能**:
- 触发条件：任务预计 > 15 分钟
- 触发时机：需求确认后

**测试驱动开发技能**:
- 触发条件：技术任务（小新负责）
- 触发时机：任务执行时
- 跳过条件：用户明确要求"跳过测试"

**增强审查技能**:
- 触发条件：Agent 完成任务后
- 触发时机：汇报给用户前
- 跳过条件：用户明确要求"跳过审查"

### 手动触发

用户可以显式要求：
- "帮我分析一下需求" → 需求澄清技能
- "写一个详细计划" → 详细计划分解技能
- "使用 TDD" → 测试驱动开发技能
- "审查代码" → 增强审查技能
```
