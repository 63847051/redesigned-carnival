## 🔄 数据流

### 标准流程
```
用户
  ↓
大领导（主控 Agent）
  ↓
需求澄清 + 任务分析
  ↓
┌─────────────────────────────────┐
│  任务复杂度评估                  │
│                                 │
│  简单（< 5 分钟）→ 直接分配      │
│  中等（5-15 分钟）→ 简单计划     │
│  复杂（> 15 分钟）→ 详细计划     │
└─────────────────────────────────┘
  ↓
分配给专家 Agent
  ↓
Agent 执行任务
  ↓
┌─────────────────────────────────┐
│  增强审查（两阶段）              │
│                                 │
│  Phase 1: 规范合规性审查         │
│  Phase 2: 代码质量审查           │
└─────────────────────────────────┘
  ↓
汇报给大领导
  ↓
汇总结果
  ↓
反馈给用户
```

## 📡 通信方式

### sessions_spawn - 异步任务
```python
# 下发任务，等结果
sessions_spawn(
    task="任务描述",
    label="任务标签",
    model="模型",
    timeout=300  # 超时时间（秒）
)
```

**用途**:
- 分发任务给子 Agent
- 异步执行，等待结果
- 并行处理多个任务

### sessions_send - Agent 间通信
```python
# 向另一个 Agent 发消息
sessions_send(
    to="agent-label",
    message="消息内容"
)
```

**用途**:
- Agent 之间直接通信
- 流水线任务（A → B → C）
- 前后台协作

### message - 向用户汇报
```python
# 向用户发送消息
message(
    action="send",
    channel="feishu",
    message="汇报内容"
)
```

**用途**:
- 汇报工作进度
- 请求用户确认
- 发送结果
