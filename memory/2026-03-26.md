## 🚨 重要教训：小新的调用方式

**问题**：
- 我总是忘记用 OpenCode CLI 调用小新
- 习惯性地用 `sessions_spawn`
- 导致使用了错误的模型

**用户的建议**：
> "你把 sessions_spawn 里的小新更新不就可以了？"

**解决方案**：
1. ✅ 更新了 `skill-isolation-rules.md` - 配置小新的正确模型
2. ✅ 更新了 `AGENTS.md` - 添加规则 3.1：小新的特殊调用规则
3. ✅ 创建了 `assign-task.sh` - 智能任务分配脚本
4. ✅ 更新了 `HEARTBEAT.md` - 添加三种调用方式

**现在有 3 种方式调用小新**：
1. **OpenCode CLI**（推荐）：
   ```bash
   opencode -m opencode/minimax-m2.5-free run "任务"
   ```

2. **智能分配脚本**（最推荐）：
   ```bash
   bash /root/.openclaw/workspace/scripts/assign-task.sh "任务" "tech"
   ```

3. **sessions_spawn**（现在也支持）：
   ```bash
   sessions_spawn -runtime subagent -model opencode/minimax-m2.5-free
   ```

**快速参考**：
```bash
# 技术任务 → 小新
opencode -m opencode/minimax-m2.5-free run "技术任务"

# 日志任务 → 小蓝
sessions_spawn -runtime subagent -model glmcode/glm-4.5-air

# 设计任务 → 设计专家
sessions_spawn -runtime subagent -model glmcode/glm-4.6
```
